# 统计每个品牌总营销天数

## 1. 题目描述

给定营销活动表 `marketing`，字段如下：

| 字段名 | 含义 |
|---|---|
| `brand` | 品牌 |
| `startdate` | 营销活动开始日期 |
| `enddate` | 营销活动结束日期 |

示例数据：

| brand | startdate | enddate |
|---|---|---|
| 华为 | 2018-08-04 | 2018-08-05 |
| 华为 | 2018-08-04 | 2020-12-25 |
| 小米 | 2018-08-15 | 2018-08-20 |
| 小米 | 2020-01-01 | 2020-01-05 |
| 苹果 | 2018-09-01 | 2018-09-05 |
| 苹果 | 2018-09-03 | 2018-09-06 |
| 苹果 | 2018-09-09 | 2018-09-15 |

要求查询：统计每个品牌的总营销天数。

> 本题口径：营销区间按闭区间计算，开始日期和结束日期都计入天数；同一品牌不同活动区间存在重叠时，重叠日期只能计算一次；如果两个活动区间不连续，中间空档日期不计入营销天数。

## 2. 期望结果

| brand | days |
|---|---:|
| 苹果 | 13 |
| 华为 | 875 |
| 小米 | 11 |

## 3. 标准答案

```sql
with data as (
    select '华为' as brand, '2018-08-04' as start_date, '2018-08-05' as end_date union all
    select '华为' as brand, '2018-08-04' as start_date, '2020-12-25' as end_date union all
    select '小米' as brand, '2018-08-15' as start_date, '2018-08-20' as end_date union all
    select '小米' as brand, '2020-01-01' as start_date, '2020-01-05' as end_date union all
    select '苹果' as brand, '2018-09-01' as start_date, '2018-09-05' as end_date union all
    select '苹果' as brand, '2018-09-03' as start_date, '2018-09-06' as end_date union all
    select '苹果' as brand, '2018-09-09' as start_date, '2018-09-15' as end_date
),
base as (
    select
        brand,
        to_date(start_date) as start_date,
        to_date(end_date) as end_date
    from data
),
expanded_data as (
    select
        brand,
        date_add(start_date, pos) as real_date
    from base
    lateral view posexplode(split(space(datediff(end_date, start_date)), ' ')) t as pos, val
)
select
    brand,
    count(distinct real_date) as days
from expanded_data
group by brand;
```

## 6. 我的复盘要点

- 【下次先想】这题虽然最后是统计天数，但核心是区间重叠去重，按项目目录应归到 `09_合并区间`。
- 【修正记法】用 `datediff(end_date, start_date)` 算区间跨度，再用 `posexplode(split(space(...), ' '))` 生成从 0 到跨度的偏移量，最后 `date_add(start_date, pos)` 展开成每天一行。
- 【本题坑点】营销活动是闭区间，开始日期和结束日期都要算；`space(datediff(...))` 配合 `posexplode` 正好能生成包含首尾的日期序列。
- 【本题坑点】重叠日期要去重，所以最终必须按品牌 `count(distinct real_date)`；如果直接把每段 `datediff + 1` 相加，华为和苹果都会把重叠天数重复计算。
- 【本题坑点】不连续区间不能补空档，例如苹果的 `2018-09-07` 和 `2018-09-08` 不在任何活动区间内，所以不能计入总营销天数。
