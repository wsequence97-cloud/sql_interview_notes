# 每个门店每日最佳补位班次

## 1. 题目描述

- 牛客题号：SQL140
- 题目：26 年携程-找出补位班次
- 难度：较难
- 原题链接：https://www.nowcoder.com/practice/ed828b0385a84e0db95f1513f43076d4
- 公开题解依据：https://www.nowcoder.com/discuss/862375369708404736

某即时零售平台会在晚高峰前向骑手发送门店补位班次的待确认邀约。同一个班次可能多次推送给不同骑手，同一骑手也可能收到同一班次的多次邀约。运营团队需要统计指定日期范围内最终成功开班的补位班次在邀约发出后 2 小时内的确认情况，并找出每个门店每天表现最好的补位班次。

给定补位班次主表 `relief_shifts`，字段如下：

| 字段名 | 类型 | 含义 |
|---|---|---|
| `shift_id` | BIGINT | 补位班次 ID，主键 |
| `store_id` | BIGINT | 门店 ID |
| `shift_date` | DATE | 班次日期 |
| `shift_period` | VARCHAR(20) | 班次时段，如 `lunch`、`afternoon`、`dinner`、`night` |
| `required_rider_count` | INT | 该班次需要补位的骑手数量 |
| `published_at` | DATETIME | 班次首次发出邀约的时间 |
| `shift_status` | VARCHAR(20) | 班次最终状态，如 `filled`、`cancelled` |

示例数据：

| shift_id | store_id | shift_date | shift_period | required_rider_count | published_at | shift_status |
|---:|---:|---|---|---:|---|---|
| 1101 | 301 | 2025-08-15 | dinner | 3 | 2025-08-15 16:00:00 | filled |
| 1102 | 301 | 2025-08-15 | night | 2 | 2025-08-15 18:30:00 | filled |
| 1103 | 302 | 2025-08-15 | dinner | 2 | 2025-08-15 15:30:00 | cancelled |
| 1104 | 301 | 2025-08-16 | lunch | 2 | 2025-08-16 09:00:00 | filled |
| 1105 | 302 | 2025-08-16 | dinner | 4 | 2025-08-16 16:10:00 | filled |

给定班次邀约反馈表 `shift_invite_responses`，字段如下：

| 字段名 | 类型 | 含义 |
|---|---|---|
| `response_id` | BIGINT | 反馈记录 ID，主键 |
| `shift_id` | BIGINT | 所属补位班次 ID，对应 `relief_shifts.shift_id` |
| `rider_id` | BIGINT | 骑手 ID |
| `invited_at` | DATETIME | 该次邀约发出时间 |
| `responded_at` | DATETIME | 骑手反馈时间；为 `NULL` 时表示未反馈 |
| `response_status` | VARCHAR(20) | 反馈结果，如 `accepted`、`declined`、`ignored` |

示例数据：

| response_id | shift_id | rider_id | invited_at | responded_at | response_status |
|---:|---:|---:|---|---|---|
| 2101 | 1101 | 9001 | 2025-08-15 16:05:00 | 2025-08-15 16:40:00 | accepted |
| 2102 | 1101 | 9002 | 2025-08-15 16:10:00 | 2025-08-15 18:20:00 | accepted |
| 2103 | 1101 | 9003 | 2025-08-15 16:20:00 | 2025-08-15 18:30:01 | accepted |
| 2104 | 1102 | 9004 | 2025-08-15 18:35:00 | 2025-08-15 19:00:00 | accepted |
| 2105 | 1102 | 9005 | 2025-08-15 18:40:00 | 2025-08-15 20:30:00 | accepted |
| 2106 | 1104 | 9001 | 2025-08-16 09:05:00 | 2025-08-16 09:50:00 | accepted |
| 2107 | 1104 | 9001 | 2025-08-16 09:20:00 | 2025-08-16 09:30:00 | accepted |
| 2108 | 1104 | 9006 | 2025-08-16 09:10:00 | 2025-08-16 11:30:00 | declined |
| 2109 | 1105 | 9007 | 2025-08-16 16:15:00 | 2025-08-16 16:50:00 | accepted |
| 2110 | 1105 | 9008 | 2025-08-16 16:20:00 | NULL | ignored |
| 2111 | 1105 | 9009 | 2025-08-16 16:25:00 | 2025-08-16 19:00:00 | accepted |

要求查询 2025-08-15 到 2025-08-17 期间，每个门店每天最终确认率最高的补位班次，返回门店 ID、班次日期、补位班次 ID、班次时段、需要补位人数、2 小时内确认成功的骑手数和 2 小时内确认率。结果按 `shift_date`、`store_id`、`shift_id` 升序排列。

> 本题口径：只统计 `shift_status = 'filled'` 的班次；同一个 `shift_id + rider_id` 只保留 `invited_at` 最早的首轮邀约；“2 小时内确认成功”要求 `response_status = 'accepted'`、`responded_at` 不为 `NULL`，且反馈时间与邀约时间之差在 0 到 7200 秒之间。确认人数按不同骑手去重计数，确认率为确认人数除以需要补位人数并四舍五入保留 2 位小数。每个 `store_id + shift_date` 按确认率降序、确认人数降序、`published_at` 升序、`shift_id` 升序依次选出一个班次。

## 3. 标准答案

```sql
WITH data AS (
    SELECT stack(
        5,
        1101, 301, '2025-08-15', 'dinner', 3, '2025-08-15 16:00:00', 'filled',
        1102, 301, '2025-08-15', 'night', 2, '2025-08-15 18:30:00', 'filled',
        1103, 302, '2025-08-15', 'dinner', 2, '2025-08-15 15:30:00', 'cancelled',
        1104, 301, '2025-08-16', 'lunch', 2, '2025-08-16 09:00:00', 'filled',
        1105, 302, '2025-08-16', 'dinner', 4, '2025-08-16 16:10:00', 'filled'
    ) AS (
        shift_id,
        store_id,
        shift_date,
        shift_period,
        required_rider_count,
        published_at,
        shift_status
    )
),
shift_invite_responses AS (
    SELECT stack(
        11,
        2101, 1101, 9001, '2025-08-15 16:05:00', '2025-08-15 16:40:00', 'accepted',
        2102, 1101, 9002, '2025-08-15 16:10:00', '2025-08-15 18:20:00', 'accepted',
        2103, 1101, 9003, '2025-08-15 16:20:00', '2025-08-15 18:30:01', 'accepted',
        2104, 1102, 9004, '2025-08-15 18:35:00', '2025-08-15 19:00:00', 'accepted',
        2105, 1102, 9005, '2025-08-15 18:40:00', '2025-08-15 20:30:00', 'accepted',
        2106, 1104, 9001, '2025-08-16 09:05:00', '2025-08-16 09:50:00', 'accepted',
        2107, 1104, 9001, '2025-08-16 09:20:00', '2025-08-16 09:30:00', 'accepted',
        2108, 1104, 9006, '2025-08-16 09:10:00', '2025-08-16 11:30:00', 'declined',
        2109, 1105, 9007, '2025-08-16 16:15:00', '2025-08-16 16:50:00', 'accepted',
        2110, 1105, 9008, '2025-08-16 16:20:00', CAST(NULL AS STRING), 'ignored',
        2111, 1105, 9009, '2025-08-16 16:25:00', '2025-08-16 19:00:00', 'accepted'
    ) AS (
        response_id,
        shift_id,
        rider_id,
        invited_at,
        responded_at,
        response_status
    )
),
relief_shifts AS (
    SELECT
        shift_id,
        store_id,
        to_date(shift_date) AS shift_date,
        shift_period,
        required_rider_count,
        CAST(published_at AS TIMESTAMP) AS published_at,
        shift_status
    FROM data
),
invite_responses AS (
    SELECT
        response_id,
        shift_id,
        rider_id,
        CAST(invited_at AS TIMESTAMP) AS invited_at,
        CAST(responded_at AS TIMESTAMP) AS responded_at,
        response_status
    FROM shift_invite_responses
),
filtered_shifts AS (
    SELECT
        shift_id,
        store_id,
        shift_date,
        shift_period,
        required_rider_count,
        published_at
    FROM relief_shifts
    WHERE shift_date BETWEEN '2025-08-15' AND '2025-08-17'
      AND shift_status = 'filled'
),
first_invite_responses AS (
    SELECT
        response_id,
        shift_id,
        rider_id,
        invited_at,
        responded_at,
        response_status,
        row_number() OVER (
            PARTITION BY shift_id, rider_id
            ORDER BY invited_at, response_id
        ) AS invite_rank
    FROM invite_responses
),
shift_acceptance_stats AS (
    SELECT
        fs.store_id,
        fs.shift_date,
        fs.shift_id,
        fs.shift_period,
        fs.required_rider_count,
        fs.published_at,
        count(DISTINCT CASE
            WHEN fir.invite_rank = 1
             AND fir.response_status = 'accepted'
             AND fir.responded_at IS NOT NULL
             AND unix_timestamp(fir.responded_at) - unix_timestamp(fir.invited_at)
                 BETWEEN 0 AND 7200
            THEN fir.rider_id
        END) AS accepted_in_2h_rider_count
    FROM filtered_shifts fs
    LEFT JOIN first_invite_responses fir
        ON fs.shift_id = fir.shift_id
    GROUP BY
        fs.store_id,
        fs.shift_date,
        fs.shift_id,
        fs.shift_period,
        fs.required_rider_count,
        fs.published_at
),
rated_shifts AS (
    SELECT
        store_id,
        shift_date,
        shift_id,
        shift_period,
        required_rider_count,
        accepted_in_2h_rider_count,
        CAST(
            round(accepted_in_2h_rider_count * 1.0 / required_rider_count, 2)
            AS DECIMAL(10, 2)
        ) AS acceptance_rate,
        published_at
    FROM shift_acceptance_stats
),
ranked_shifts AS (
    SELECT
        store_id,
        shift_date,
        shift_id,
        shift_period,
        required_rider_count,
        accepted_in_2h_rider_count,
        acceptance_rate,
        row_number() OVER (
            PARTITION BY store_id, shift_date
            ORDER BY
                acceptance_rate DESC,
                accepted_in_2h_rider_count DESC,
                published_at ASC,
                shift_id ASC
        ) AS shift_rank
    FROM rated_shifts
)
SELECT
    store_id,
    shift_date,
    shift_id,
    shift_period,
    required_rider_count,
    accepted_in_2h_rider_count,
    acceptance_rate
FROM ranked_shifts
WHERE shift_rank = 1
ORDER BY shift_date, store_id, shift_id;
```
