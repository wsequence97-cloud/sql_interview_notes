# 某平台七日留存计算

## 1. 题目描述

给定表 `data`，记录用户在平台上的访问时间，字段如下：

| 字段名 | 含义 |
|---|---|
| `uid` | 用户 ID |
| `datetime` | 用户访问时间 |

示例数据：

| uid | datetime |
|---:|---|
| 1 | 2019-01-01 00:00:00 |
| 1 | 2019-01-01 01:00:00 |
| 1 | 2019-01-02 00:00:00 |
| 1 | 2019-01-04 00:00:00 |
| 1 | 2019-01-08 00:00:00 |
| 1 | 2019-01-09 00:00:00 |
| 2 | 2019-02-01 00:00:00 |
| 2 | 2019-02-02 00:00:00 |
| 2 | 2019-02-08 00:00:00 |
| 2 | 2019-02-10 00:00:00 |
| 3 | 2019-03-04 00:00:00 |
| 3 | 2019-03-05 00:00:00 |
| 3 | 2019-03-07 00:00:00 |
| 3 | 2019-03-11 00:00:00 |
| 3 | 2019-03-12 00:00:00 |

要求查询：按用户活跃日期统计该日期的次日留存率、三日留存率和七日留存率。

> 本题口径：同一用户同一天多次访问只算一个活跃用户；N 日留存表示用户在基准日期活跃后，第 N 天再次活跃。

## 3. 标准答案

```sql
with data as (
    select stack(
        15,
        1, '2019-01-01 00:00:00',
        1, '2019-01-01 01:00:00',
        1, '2019-01-02 00:00:00',
        1, '2019-01-04 00:00:00',
        1, '2019-01-08 00:00:00',
        1, '2019-01-09 00:00:00',
        2, '2019-02-01 00:00:00',
        2, '2019-02-02 00:00:00',
        2, '2019-02-08 00:00:00',
        2, '2019-02-10 00:00:00',
        3, '2019-03-04 00:00:00',
        3, '2019-03-05 00:00:00',
        3, '2019-03-07 00:00:00',
        3, '2019-03-11 00:00:00',
        3, '2019-03-12 00:00:00'
    ) as (uid, `datetime`)
),
active_distinct as (
    select distinct
        uid,
        to_date(`datetime`) as active_date
    from data
),
retention_data as (
    select
        a.active_date as `date`,
        count(distinct a.uid) as active_user_cnt,
        count(distinct case when datediff(b.active_date, a.active_date) = 1 then a.uid end) as lst1date_user_cnt,
        count(distinct case when datediff(b.active_date, a.active_date) = 3 then a.uid end) as lst3date_user_cnt,
        count(distinct case when datediff(b.active_date, a.active_date) = 7 then a.uid end) as lst7date_user_cnt
    from active_distinct a
    left join active_distinct b
        on a.uid = b.uid
    group by a.active_date
)
select
    `date`,
    lst1date_user_cnt * 1.0 / active_user_cnt as lst1date_rate,
    lst3date_user_cnt * 1.0 / active_user_cnt as lst3date_rate,
    lst7date_user_cnt * 1.0 / active_user_cnt as lst7date_rate
from retention_data
order by `date`;
```

    date,

    cnt_uid_7 / cnt_uid_all

    retention_data

## 6. 我的复盘要点

- 【本题坑点】同一用户同一天可能访问多次，所以要先按 `uid + active_date` 去重，否则 2019-01-01 的用户 1 会被重复计入。
- 【错误原因】我原 SQL 写了 `active_distinct`，但后面又回到原始 `data` 表计算，导致同一天多次访问没有真正去重。
- 【错误原因】原 SQL 按 `uid` 分组不对，本题要按基准活跃日期分组；否则算出来的是用户维度，不是日期维度的留存率。
- 【修正记法】N 日留存就是把基准活跃日期表 `a` 和后续活跃日期表 `b` 按用户关联，再用 `datediff(b.active_date, a.active_date) = N` 判断。
- 【错误原因】Hive 的 `join on` 里不要写 `a.active_date < b.active_date` 这种同时引用左右表的非等值条件；本题直接按 `uid` 关联，再在 `case when datediff(...) = N` 中判断即可。
