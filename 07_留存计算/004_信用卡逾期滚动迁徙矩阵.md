# 26年某银行-信用卡逾期滚动迁徙矩阵

## 1. 题目描述

- 牛客题号：SQL159
- 难度：困难
- 题目来源：[26年某银行-信用卡逾期滚动迁徙矩阵](https://www.nowcoder.com/practice/1e3a3dad8fb64a48924eb28d5abf2b86)
- 题解依据：[题解｜26年某银行-信用卡逾期滚动迁徙矩阵](https://www.nowcoder.com/discuss/905118757662511104)

银行信用卡风控团队每月观察账户逾期状态从 M0、M1、M2、M3+ 到核销的迁徙情况。迁徙矩阵用于判断资产质量是否恶化，以及核销对余额的影响。

给定信用卡账户表 `card_accounts`，字段如下：

| 字段名 | 类型 | 含义 |
|---|---|---|
| `account_id` | VARCHAR(20) | 信用卡账户 ID |
| `customer_id` | VARCHAR(20) | 客户 ID |
| `open_date` | DATE | 开户日期 |

示例数据：

| account_id | customer_id | open_date |
|---|---|---|
| A1 | C1 | 2025-01-01 |
| A2 | C2 | 2025-01-01 |
| A3 | C3 | 2025-01-01 |

给定月账单表 `monthly_bills`，字段如下，其中 `bill_month` 为每月第一天：

| 字段名 | 类型 | 含义 |
|---|---|---|
| `bill_id` | INT | 账单 ID |
| `account_id` | VARCHAR(20) | 信用卡账户 ID |
| `bill_month` | DATE | 账单月份 |
| `due_date` | DATE | 到期还款日 |
| `bill_amount` | DECIMAL(12,2) | 账单金额 |

示例数据：

| bill_id | account_id | bill_month | due_date | bill_amount |
|---:|---|---|---|---:|
| 1 | A1 | 2026-01-01 | 2026-01-20 | 1000.00 |
| 2 | A1 | 2026-02-01 | 2026-02-20 | 1200.00 |
| 3 | A2 | 2026-01-01 | 2025-12-15 | 800.00 |
| 4 | A2 | 2026-02-01 | 2025-12-15 | 900.00 |
| 5 | A3 | 2026-01-01 | 2026-01-10 | 500.00 |
| 6 | A3 | 2026-02-01 | 2026-01-10 | 500.00 |

给定还款表 `repayments`，字段如下：

| 字段名 | 类型 | 含义 |
|---|---|---|
| `repay_id` | INT | 还款记录 ID |
| `bill_id` | INT | 账单 ID |
| `pay_date` | DATE | 还款日期 |
| `amount` | DECIMAL(12,2) | 还款金额 |

示例数据：

| repay_id | bill_id | pay_date | amount |
|---:|---:|---|---:|
| 1 | 1 | 2026-01-19 | 1000.00 |
| 2 | 2 | 2026-02-25 | 1200.00 |
| 3 | 3 | 2025-12-20 | 200.00 |
| 4 | 5 | 2026-01-09 | 200.00 |
| 5 | 5 | 2026-01-11 | 300.00 |

给定核销表 `writeoffs`，字段如下，其中 `writeoff_month` 为核销月份第一天：

| 字段名 | 类型 | 含义 |
|---|---|---|
| `account_id` | VARCHAR(20) | 信用卡账户 ID |
| `writeoff_month` | DATE | 核销月份 |

示例数据：

| account_id | writeoff_month |
|---|---|
| A2 | 2026-02-01 |

对每张账单，统计该 `bill_id` 在 `due_date` 当天及之前的还款总额，并按以下规则生成月度状态：

- 若账户在 `writeoff_month` 当月或之前已核销，该月状态为 `WRITE_OFF`。
- 否则，到期日前还款总额大于等于账单金额时，状态为 `M0`。
- 未足额还款时，用 `LAST_DAY(bill_month)` 与 `due_date` 的天数差判断：1 到 30 天为 `M1`，31 到 60 天为 `M2`，超过 60 天为 `M3P`。
- 账单余额 `balance = bill_amount - 到期日前还款总额`，若为负数则按 0 处理。

只统计相邻月份均存在账单快照的账户迁徙。迁徙率等于当前 `from_bucket` 在该起始月份的迁出账户数除以该 `from_bucket` 在该起始月份的总账户数，保留 4 位小数。

`movement` 将 `WRITE_OFF` 视为最差状态，其余状态按 `M0`、`M1`、`M2`、`M3P` 从好到坏排序：变差为 `worsen`，变好为 `improve`，不变为 `stable`。

要求输出 `month_from`、`from_bucket`、`to_bucket`、`account_count`、`total_balance`、`migration_rate`、`movement`。其中 `month_from` 格式为 `YYYY-MM`，`total_balance` 为起始月份这些账户的余额合计。结果按 `month_from`、`from_bucket`、`to_bucket` 升序排列。

## 3. 标准答案

以下答案根据公开题解的“到期日前还款聚合、月度状态划分、相邻月份关联”思路改写为 Hive SQL，并按原题定义计算迁徙率分母和状态变化方向。

```sql
WITH data AS (
    SELECT stack(
        15,
        'card_accounts', 'A1', 'C1', '2025-01-01', '',  '',           '',           '',        '',  '',           '',        '',
        'card_accounts', 'A2', 'C2', '2025-01-01', '',  '',           '',           '',        '',  '',           '',        '',
        'card_accounts', 'A3', 'C3', '2025-01-01', '',  '',           '',           '',        '',  '',           '',        '',
        'monthly_bills', 'A1', '',   '',           '1', '2026-01-01', '2026-01-20', '1000.00', '',  '',           '',        '',
        'monthly_bills', 'A1', '',   '',           '2', '2026-02-01', '2026-02-20', '1200.00', '',  '',           '',        '',
        'monthly_bills', 'A2', '',   '',           '3', '2026-01-01', '2025-12-15', '800.00',  '',  '',           '',        '',
        'monthly_bills', 'A2', '',   '',           '4', '2026-02-01', '2025-12-15', '900.00',  '',  '',           '',        '',
        'monthly_bills', 'A3', '',   '',           '5', '2026-01-01', '2026-01-10', '500.00',  '',  '',           '',        '',
        'monthly_bills', 'A3', '',   '',           '6', '2026-02-01', '2026-01-10', '500.00',  '',  '',           '',        '',
        'repayments',    '',   '',   '',           '1', '',           '',           '',        '1', '2026-01-19', '1000.00', '',
        'repayments',    '',   '',   '',           '2', '',           '',           '',        '2', '2026-02-25', '1200.00', '',
        'repayments',    '',   '',   '',           '3', '',           '',           '',        '3', '2025-12-20', '200.00',  '',
        'repayments',    '',   '',   '',           '5', '',           '',           '',        '4', '2026-01-09', '200.00',  '',
        'repayments',    '',   '',   '',           '5', '',           '',           '',        '5', '2026-01-11', '300.00',  '',
        'writeoffs',     'A2', '',   '',           '',  '',           '',           '',        '',  '',           '',        '2026-02-01'
    ) AS (
        source_table,
        account_id,
        customer_id,
        open_date,
        bill_id,
        bill_month,
        due_date,
        bill_amount,
        repay_id,
        pay_date,
        amount,
        writeoff_month
    )
),
card_accounts AS (
    SELECT
        account_id,
        customer_id,
        to_date(open_date) AS open_date
    FROM data
    WHERE source_table = 'card_accounts'
),
monthly_bills AS (
    SELECT
        CAST(bill_id AS INT) AS bill_id,
        account_id,
        to_date(bill_month) AS bill_month,
        to_date(due_date) AS due_date,
        CAST(bill_amount AS DECIMAL(12, 2)) AS bill_amount
    FROM data
    WHERE source_table = 'monthly_bills'
),
repayments AS (
    SELECT
        CAST(repay_id AS INT) AS repay_id,
        CAST(bill_id AS INT) AS bill_id,
        to_date(pay_date) AS pay_date,
        CAST(amount AS DECIMAL(12, 2)) AS amount
    FROM data
    WHERE source_table = 'repayments'
),
writeoffs AS (
    SELECT
        account_id,
        to_date(writeoff_month) AS writeoff_month
    FROM data
    WHERE source_table = 'writeoffs'
),
bill_payment AS (
    SELECT
        b.bill_id,
        b.account_id,
        b.bill_month,
        b.due_date,
        b.bill_amount,
        CAST(
            coalesce(
                sum(
                    CASE
                        WHEN r.pay_date <= b.due_date THEN r.amount
                        ELSE CAST(0 AS DECIMAL(12, 2))
                    END
                ),
                CAST(0 AS DECIMAL(12, 2))
            ) AS DECIMAL(12, 2)
        ) AS paid_by_due
    FROM monthly_bills b
    LEFT JOIN repayments r
        ON b.bill_id = r.bill_id
    GROUP BY
        b.bill_id,
        b.account_id,
        b.bill_month,
        b.due_date,
        b.bill_amount
),
monthly_snapshot AS (
    SELECT
        b.bill_id,
        b.account_id,
        b.bill_month,
        CAST(
            greatest(
                b.bill_amount - b.paid_by_due,
                CAST(0 AS DECIMAL(12, 2))
            ) AS DECIMAL(12, 2)
        ) AS balance,
        CASE
            WHEN w.writeoff_month IS NOT NULL
                AND b.bill_month >= w.writeoff_month THEN 'WRITE_OFF'
            WHEN b.paid_by_due >= b.bill_amount THEN 'M0'
            WHEN datediff(last_day(b.bill_month), b.due_date) BETWEEN 1 AND 30 THEN 'M1'
            WHEN datediff(last_day(b.bill_month), b.due_date) BETWEEN 31 AND 60 THEN 'M2'
            ELSE 'M3P'
        END AS bucket
    FROM bill_payment b
    LEFT JOIN writeoffs w
        ON b.account_id = w.account_id
),
ranked_snapshot AS (
    SELECT
        bill_id,
        account_id,
        bill_month,
        balance,
        bucket,
        CASE bucket
            WHEN 'M0' THEN 0
            WHEN 'M1' THEN 1
            WHEN 'M2' THEN 2
            WHEN 'M3P' THEN 3
            WHEN 'WRITE_OFF' THEN 4
        END AS bucket_rank
    FROM monthly_snapshot
),
bucket_totals AS (
    SELECT
        bill_month,
        bucket,
        count(DISTINCT account_id) AS from_account_count
    FROM ranked_snapshot
    GROUP BY bill_month, bucket
),
adjacent_transitions AS (
    SELECT
        f.bill_month AS month_from,
        f.bucket AS from_bucket,
        t.bucket AS to_bucket,
        f.bucket_rank AS from_bucket_rank,
        t.bucket_rank AS to_bucket_rank,
        f.account_id,
        f.balance
    FROM ranked_snapshot f
    JOIN ranked_snapshot t
        ON f.account_id = t.account_id
        AND t.bill_month = add_months(f.bill_month, 1)
),
transition_summary AS (
    SELECT
        month_from,
        from_bucket,
        to_bucket,
        from_bucket_rank,
        to_bucket_rank,
        count(DISTINCT account_id) AS account_count,
        CAST(sum(balance) AS DECIMAL(18, 2)) AS total_balance
    FROM adjacent_transitions
    GROUP BY
        month_from,
        from_bucket,
        to_bucket,
        from_bucket_rank,
        to_bucket_rank
)
SELECT
    date_format(s.month_from, 'yyyy-MM') AS month_from,
    s.from_bucket,
    s.to_bucket,
    s.account_count,
    s.total_balance,
    CAST(
        round(s.account_count * 1.0 / b.from_account_count, 4)
        AS DECIMAL(10, 4)
    ) AS migration_rate,
    CASE
        WHEN s.from_bucket_rank < s.to_bucket_rank THEN 'worsen'
        WHEN s.from_bucket_rank > s.to_bucket_rank THEN 'improve'
        ELSE 'stable'
    END AS movement
FROM transition_summary s
JOIN bucket_totals b
    ON s.month_from = b.bill_month
    AND s.from_bucket = b.bucket
ORDER BY month_from, from_bucket, to_bucket;
```
