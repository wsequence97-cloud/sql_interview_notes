# 25年得物-潮鞋新品发售后 N 日复购留存矩阵

## 1. 题目描述

题目来源：牛客 SQL146（中等），[25年得物-潮鞋新品发售后 N 日复购留存矩阵](https://www.nowcoder.com/practice/1a4a359d3a524cc0830c52985888bd38)。公开解法依据：[牛客题解](https://www.nowcoder.com/discuss/884019944399155200)。

某潮流电商平台运营团队需要分析新品系列发售后的“首购 -> 复购”留存情况。以用户在某系列的首次下单日为 Day 0，统计其在 Day+1、Day+3、Day+7 和 Day+14 当天是否再次购买同一系列。

给定 `t_series`（潮鞋系列字典）表，字段如下：

| 字段名 | 类型 | 含义 |
|---|---|---|
| `series_id` | `BIGINT` | 系列 ID |
| `series_name` | `VARCHAR(64)` | 系列名称 |
| `launch_date` | `DATE` | 系列首发日期，位于 2025 年内 |

示例数据：

| series_id | series_name | launch_date |
|---:|---|---|
| 1 | AJ1 Retro High | 2025-03-01 |
| 2 | Yeezy 350 V2 | 2025-03-10 |
| 3 | Dunk Low Panda | 2025-04-01 |

给定 `t_order`（订单明细）表，字段如下：

| 字段名 | 类型 | 含义 |
|---|---|---|
| `order_id` | `BIGINT` | 订单 ID |
| `user_id` | `BIGINT` | 用户 ID |
| `series_id` | `BIGINT` | 所购商品所属系列 ID，关联 `t_series.series_id` |
| `pay_amount` | `DECIMAL(10,2)` | 实付金额 |
| `pay_time` | `DATETIME` | 支付时间，位于 2025 年内，且已剔除未支付订单 |

示例数据：

| order_id | user_id | series_id | pay_amount | pay_time |
|---:|---:|---:|---:|---|
| 10001 | 1001 | 1 | 1299.00 | 2025-03-02 10:15:00 |
| 10002 | 1001 | 1 | 1399.00 | 2025-03-03 20:00:00 |
| 10003 | 1001 | 1 | 1199.00 | 2025-03-09 09:30:00 |
| 10004 | 1002 | 1 | 1299.00 | 2025-03-02 11:00:00 |
| 10005 | 1002 | 1 | 1259.00 | 2025-03-16 14:00:00 |
| 10006 | 1003 | 1 | 1299.00 | 2025-03-05 08:00:00 |
| 10007 | 1004 | 2 | 999.00 | 2025-03-11 12:00:00 |
| 10008 | 1004 | 2 | 1059.00 | 2025-03-14 12:00:00 |

要求统计每个系列的首购用户留存矩阵，输出系列 ID、系列名称、首购用户数，以及 Day+1、Day+3、Day+7、Day+14 的复购留存率。

> 本题口径：每个用户在每个系列下最早一笔订单的下单日为该用户的 Day 0；复购指除首购订单外，该用户再次购买同一系列。各留存日按 `DATE(pay_time)` 对齐的日历日计算，不按 24 小时滚动窗口计算。留存率为指定留存日有复购的首购用户数除以首购用户数，四舍五入保留 2 位小数；无首购用户的系列不输出。结果按 `d7_rate` 降序、`first_buyer_cnt` 降序、`series_id` 升序排列。

## 3. 标准答案

```sql
WITH data AS (
    SELECT stack(
        3,
        CAST(1 AS BIGINT), 'AJ1 Retro High', '2025-03-01',
        CAST(2 AS BIGINT), 'Yeezy 350 V2', '2025-03-10',
        CAST(3 AS BIGINT), 'Dunk Low Panda', '2025-04-01'
    ) AS (series_id, series_name, launch_date)
),
t_series AS (
    SELECT
        series_id,
        series_name,
        to_date(launch_date) AS launch_date
    FROM data
),
t_order AS (
    SELECT stack(
        8,
        CAST(10001 AS BIGINT), CAST(1001 AS BIGINT), CAST(1 AS BIGINT), CAST(1299.00 AS DECIMAL(10, 2)), '2025-03-02 10:15:00',
        CAST(10002 AS BIGINT), CAST(1001 AS BIGINT), CAST(1 AS BIGINT), CAST(1399.00 AS DECIMAL(10, 2)), '2025-03-03 20:00:00',
        CAST(10003 AS BIGINT), CAST(1001 AS BIGINT), CAST(1 AS BIGINT), CAST(1199.00 AS DECIMAL(10, 2)), '2025-03-09 09:30:00',
        CAST(10004 AS BIGINT), CAST(1002 AS BIGINT), CAST(1 AS BIGINT), CAST(1299.00 AS DECIMAL(10, 2)), '2025-03-02 11:00:00',
        CAST(10005 AS BIGINT), CAST(1002 AS BIGINT), CAST(1 AS BIGINT), CAST(1259.00 AS DECIMAL(10, 2)), '2025-03-16 14:00:00',
        CAST(10006 AS BIGINT), CAST(1003 AS BIGINT), CAST(1 AS BIGINT), CAST(1299.00 AS DECIMAL(10, 2)), '2025-03-05 08:00:00',
        CAST(10007 AS BIGINT), CAST(1004 AS BIGINT), CAST(2 AS BIGINT), CAST(999.00 AS DECIMAL(10, 2)), '2025-03-11 12:00:00',
        CAST(10008 AS BIGINT), CAST(1004 AS BIGINT), CAST(2 AS BIGINT), CAST(1059.00 AS DECIMAL(10, 2)), '2025-03-14 12:00:00'
    ) AS (order_id, user_id, series_id, pay_amount, pay_time)
),
base_order AS (
    SELECT
        order_id,
        series_id,
        user_id,
        to_date(pay_time) AS pay_date,
        MIN(to_date(pay_time)) OVER (
            PARTITION BY series_id, user_id
        ) AS first_pay_date,
        ROW_NUMBER() OVER (
            PARTITION BY series_id, user_id
            ORDER BY pay_time, order_id
        ) AS rn
    FROM t_order
),
user_flags AS (
    SELECT
        series_id,
        user_id,
        MAX(CASE WHEN rn > 1 AND datediff(pay_date, first_pay_date) = 1 THEN 1 ELSE 0 END) AS d1_flag,
        MAX(CASE WHEN rn > 1 AND datediff(pay_date, first_pay_date) = 3 THEN 1 ELSE 0 END) AS d3_flag,
        MAX(CASE WHEN rn > 1 AND datediff(pay_date, first_pay_date) = 7 THEN 1 ELSE 0 END) AS d7_flag,
        MAX(CASE WHEN rn > 1 AND datediff(pay_date, first_pay_date) = 14 THEN 1 ELSE 0 END) AS d14_flag
    FROM base_order
    GROUP BY series_id, user_id
),
series_metrics AS (
    SELECT
        series_id,
        COUNT(*) AS first_buyer_cnt,
        SUM(d1_flag) * 1.0 / COUNT(*) AS d1_rate,
        SUM(d3_flag) * 1.0 / COUNT(*) AS d3_rate,
        SUM(d7_flag) * 1.0 / COUNT(*) AS d7_rate,
        SUM(d14_flag) * 1.0 / COUNT(*) AS d14_rate
    FROM user_flags
    GROUP BY series_id
),
final_result AS (
    SELECT
        sm.series_id,
        ts.series_name,
        sm.first_buyer_cnt,
        CAST(ROUND(sm.d1_rate, 2) AS DECIMAL(10, 2)) AS d1_rate,
        CAST(ROUND(sm.d3_rate, 2) AS DECIMAL(10, 2)) AS d3_rate,
        CAST(ROUND(sm.d7_rate, 2) AS DECIMAL(10, 2)) AS d7_rate,
        CAST(ROUND(sm.d14_rate, 2) AS DECIMAL(10, 2)) AS d14_rate
    FROM series_metrics sm
    INNER JOIN t_series ts
        ON sm.series_id = ts.series_id
)
SELECT
    series_id,
    series_name,
    first_buyer_cnt,
    d1_rate,
    d3_rate,
    d7_rate,
    d14_rate
FROM final_result
ORDER BY
    d7_rate DESC,
    first_buyer_cnt DESC,
    series_id ASC;
```
