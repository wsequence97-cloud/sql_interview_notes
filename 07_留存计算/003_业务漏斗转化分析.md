# 业务漏斗转化分析

## 1. 题目描述

题目来源：[牛客 SQL153：25年京东-漏斗](https://www.nowcoder.com/practice/f97762a1c1c44f5ab5e4b6d6a1f42066)。难度：较难。

某调香工作室想了解客户的“嗅觉旅程”：客户从第一瓶香水开始，是会深耕同一家族，还是会主动探索新家族。工作室定义了一个 4 步漏斗，按客户首单香水家族分组统计每一步的转化率。

漏斗严格保序：客户必须先达到上一步，才能进入下一步统计，后一步的客户必然是前一步客户的子集。

`t_perfume` 为香水产品档案表，每款香水一行，字段如下：

| 字段名 | 类型 | 含义 |
|---|---|---|
| `perfume_id` | `BIGINT` | 香水编号，主键 |
| `perfume_name` | `VARCHAR(64)` | 香水名称 |
| `family` | `VARCHAR(16)` | 香水家族，取值为 `floral`、`woody`、`citrus`、`oriental`、`fresh`、`gourmand` |
| `price` | `DECIMAL(7,2)` | 标价（元） |

`t_perfume` 示例数据：

| perfume_id | perfume_name | family | price |
|---:|---|---|---:|
| 1 | Rose Garden | floral | 380.00 |
| 2 | Jasmine Night | floral | 420.00 |
| 3 | Sandalwood Echo | woody | 560.00 |
| 4 | Cedar Storm | woody | 480.00 |
| 5 | Lemon Zest | citrus | 220.00 |
| 6 | Bergamot Sky | citrus | 260.00 |
| 7 | Amber Whisper | oriental | 680.00 |

`t_purchase` 为购买流水表，每笔购买一行，字段如下：

| 字段名 | 类型 | 含义 |
|---|---|---|
| `purchase_id` | `BIGINT` | 购买编号，主键 |
| `customer_id` | `BIGINT` | 客户编号 |
| `perfume_id` | `BIGINT` | 香水编号，关联 `t_perfume.perfume_id` |
| `purchase_date` | `DATE` | 购买日期 |
| `qty` | `TINYINT` | 购买数量 |
| `pay_amount` | `DECIMAL(8,2)` | 实付金额 |

漏斗步骤定义如下，每位客户单独判定：

1. D0 首单：客户在 2025 年内的第一笔购买，按 `purchase_date` 升序取最早一单；若同一天有多单，取 `purchase_id` 最小的一笔。首单对应的 `family` 称为该客户的 `first_family`。
2. Step 2 同家族复购：在 `purchase_date` 位于 D0+1 至 D0+30（含边界，即 1～30 天）的窗口内，存在至少一笔购买，其 `family = first_family`。
3. Step 3 异家族扩展：在 `purchase_date` 位于 D0+31 至 D0+60（含边界）的窗口内，存在至少一笔购买，其 `family <> first_family`；客户必须先满足 Step 2。
4. Step 4 三家族覆盖：在 `purchase_date` 位于 D0 至 D0+90（含首单及边界）的窗口内，客户累计接触的不同 `family` 数量不少于 3；客户必须先满足 Step 3。

只考虑 `purchase_date` 在 2025-01-01 至 2025-12-31（含边界）之间的购买。按 `first_family` 分组，只输出 `step1_first_buyer_cnt > 0` 的家族，输出字段如下：

| 字段名 | 含义 |
|---|---|
| `first_family` | 首单家族 |
| `step1_first_buyer_cnt` | 该家族作为客户首单家族的客户人数 |
| `step2_same_family_repeat_cnt` | 满足 Step 2 的客户人数 |
| `step3_cross_family_cnt` | 同时满足 Step 2 和 Step 3 的客户人数 |
| `step4_three_family_cnt` | 同时满足 Step 2、Step 3 和 Step 4 的客户人数 |
| `step2_rate` | Step 2 / Step 1 × 100，四舍五入保留 2 位；分母为 0 时输出 0.00 |
| `step3_rate` | Step 3 / Step 2 × 100，四舍五入保留 2 位；分母为 0 时输出 0.00 |
| `step4_rate` | Step 4 / Step 3 × 100，四舍五入保留 2 位；分母为 0 时输出 0.00 |

排序规则：先按 `step1_first_buyer_cnt` 降序，再按 `first_family` 字典序升序。

样例中各客户的漏斗判定如下：

| customer | first_family | D0 | step2 | step3 | step4 |
|---:|---|---|---|---|---|
| 1001 | floral | 2025-03-01 | ✓（15 天 Jasmine） | ✓（45 天 Sandalwood） | ✓（75 天 Lemon，3 家族） |
| 1002 | floral | 2025-04-01 | ✓（20 天 Jasmine） | ✓（50 天 Cedar） | ✗（90 天内只 2 家族） |
| 1003 | floral | 2025-05-01 | ✓（10 天 Jasmine） | ✗ | — |
| 1004 | floral | 2025-06-01 | ✗ | — | — |
| 1005 | woody | 2025-03-15 | ✓（20 天 Cedar） | ✓（50 天 Rose） | ✓（80 天 Lemon，3 家族） |
| 1006 | woody | 2025-04-10 | ✗（30 天内无 woody 复购） | — | — |
| 1007 | citrus | 2025-07-01 | ✓（15 天 Bergamot） | ✗（无异家族购买） | — |
| 1008 | oriental | 2025-08-01 | ✗ | — | — |

## 3. 标准答案

参考[公开题解](https://www.nowcoder.com/discuss/902608440495783936)的分步漏斗思路，转换为以下完整 Hive SQL：

```sql
WITH data AS (
    select stack(
        19,
        101, 1001, 1, '2025-03-01', 1, cast(380.00 as decimal(8, 2)),
        102, 1001, 2, '2025-03-16', 1, cast(420.00 as decimal(8, 2)),
        103, 1001, 3, '2025-04-15', 1, cast(560.00 as decimal(8, 2)),
        104, 1001, 5, '2025-05-15', 1, cast(220.00 as decimal(8, 2)),
        105, 1002, 1, '2025-04-01', 1, cast(380.00 as decimal(8, 2)),
        106, 1002, 2, '2025-04-21', 1, cast(420.00 as decimal(8, 2)),
        107, 1002, 4, '2025-05-21', 1, cast(480.00 as decimal(8, 2)),
        108, 1003, 1, '2025-05-01', 1, cast(380.00 as decimal(8, 2)),
        109, 1003, 2, '2025-05-11', 1, cast(420.00 as decimal(8, 2)),
        110, 1004, 1, '2025-06-01', 1, cast(380.00 as decimal(8, 2)),
        111, 1005, 3, '2025-03-15', 1, cast(560.00 as decimal(8, 2)),
        112, 1005, 4, '2025-04-04', 1, cast(480.00 as decimal(8, 2)),
        113, 1005, 1, '2025-05-04', 1, cast(380.00 as decimal(8, 2)),
        114, 1005, 5, '2025-06-03', 1, cast(220.00 as decimal(8, 2)),
        115, 1006, 3, '2025-04-10', 1, cast(560.00 as decimal(8, 2)),
        116, 1006, 1, '2025-05-20', 1, cast(380.00 as decimal(8, 2)),
        117, 1007, 5, '2025-07-01', 1, cast(220.00 as decimal(8, 2)),
        118, 1007, 6, '2025-07-16', 1, cast(260.00 as decimal(8, 2)),
        119, 1008, 7, '2025-08-01', 1, cast(680.00 as decimal(8, 2))
    ) as (purchase_id, customer_id, perfume_id, purchase_date, qty, pay_amount)
),
perfume_data as (
    select stack(
        7,
        1, 'Rose Garden',      'floral',   cast(380.00 as decimal(7, 2)),
        2, 'Jasmine Night',    'floral',   cast(420.00 as decimal(7, 2)),
        3, 'Sandalwood Echo',  'woody',    cast(560.00 as decimal(7, 2)),
        4, 'Cedar Storm',      'woody',    cast(480.00 as decimal(7, 2)),
        5, 'Lemon Zest',       'citrus',   cast(220.00 as decimal(7, 2)),
        6, 'Bergamot Sky',     'citrus',   cast(260.00 as decimal(7, 2)),
        7, 'Amber Whisper',    'oriental', cast(680.00 as decimal(7, 2))
    ) as (perfume_id, perfume_name, family, price)
),
purchase_base as (
    select
        d.purchase_id,
        d.customer_id,
        d.perfume_id,
        to_date(d.purchase_date) as purchase_date,
        d.qty,
        d.pay_amount,
        p.perfume_name,
        p.family,
        p.price
    from data d
    join perfume_data p
        on d.perfume_id = p.perfume_id
    where to_date(d.purchase_date) between to_date('2025-01-01') and to_date('2025-12-31')
),
ranked_purchase as (
    select
        purchase_id,
        customer_id,
        perfume_id,
        purchase_date,
        family,
        row_number() over (
            partition by customer_id
            order by purchase_date, purchase_id
        ) as purchase_rank
    from purchase_base
),
first_order as (
    select
        customer_id,
        purchase_date as d0,
        family as first_family
    from ranked_purchase
    where purchase_rank = 1
),
customer_metrics as (
    select
        f.customer_id,
        f.first_family,
        f.d0,
        max(
            case
                when p.purchase_date between date_add(f.d0, 1) and date_add(f.d0, 30)
                    and p.family = f.first_family
                then 1 else 0
            end
        ) as step2_flag,
        max(
            case
                when p.purchase_date between date_add(f.d0, 31) and date_add(f.d0, 60)
                    and p.family <> f.first_family
                then 1 else 0
            end
        ) as step3_candidate_flag,
        count(
            distinct case
                when p.purchase_date between f.d0 and date_add(f.d0, 90)
                then p.family
            end
        ) as family_cnt_90d
    from first_order f
    join purchase_base p
        on f.customer_id = p.customer_id
    group by
        f.customer_id,
        f.first_family,
        f.d0
),
funnel_flags as (
    select
        customer_id,
        first_family,
        step2_flag,
        case
            when step2_flag = 1 and step3_candidate_flag = 1 then 1
            else 0
        end as step3_flag,
        case
            when step2_flag = 1
                and step3_candidate_flag = 1
                and family_cnt_90d >= 3
            then 1 else 0
        end as step4_flag
    from customer_metrics
),
family_counts as (
    select
        first_family,
        count(*) as step1_first_buyer_cnt,
        sum(step2_flag) as step2_same_family_repeat_cnt,
        sum(step3_flag) as step3_cross_family_cnt,
        sum(step4_flag) as step4_three_family_cnt
    from funnel_flags
    group by first_family
)
select
    first_family,
    step1_first_buyer_cnt,
    step2_same_family_repeat_cnt,
    step3_cross_family_cnt,
    step4_three_family_cnt,
    cast(
        case
            when step1_first_buyer_cnt = 0 then 0.00
            else round(step2_same_family_repeat_cnt * 100.0 / step1_first_buyer_cnt, 2)
        end as decimal(10, 2)
    ) as step2_rate,
    cast(
        case
            when step2_same_family_repeat_cnt = 0 then 0.00
            else round(step3_cross_family_cnt * 100.0 / step2_same_family_repeat_cnt, 2)
        end as decimal(10, 2)
    ) as step3_rate,
    cast(
        case
            when step3_cross_family_cnt = 0 then 0.00
            else round(step4_three_family_cnt * 100.0 / step3_cross_family_cnt, 2)
        end as decimal(10, 2)
    ) as step4_rate
from family_counts
where step1_first_buyer_cnt > 0
order by
    step1_first_buyer_cnt desc,
    first_family asc;
```
