# 统计原创文章被引用次数

## 1. 题目描述

给定文章表 `article`，字段如下：

| 字段名 | 含义 |
|---|---|
| `id` | 文章 ID |
| `oid` | 引用的原创文章 ID，`oid = 0` 表示该文章为原创 |

示例数据：

| id | oid |
|---:|---:|
| 1 | 0 |
| 2 | 0 |
| 3 | 1 |
| 4 | 1 |
| 5 | 2 |
| 6 | 0 |

要求查询：不用 `join`，统计原创文章被引用的次数。

> 本题口径：只有 `oid` 指向原创文章时才计为有效引用；`oid = 0` 的原创文章本身不算引用记录。没有被引用的原创文章不输出。

## 2. 期望结果

| oid | nums |
|---:|---:|
| 1 | 2 |
| 2 | 1 |

## 3. 标准答案

```sql
with article as (
    select stack(
        6,
        1, 0,
        2, 0,
        3, 1,
        4, 1,
        5, 2,
        6, 0
    ) as (id, oid)
),
marked_data as (
    select
        id,
        oid,
        collect_set(if(oid = 0, id, null)) over () as origin_ids
    from article
)
select
    oid,
    sum(if(array_contains(origin_ids, oid), 1, 0)) as nums
from marked_data
where oid <> 0
group by oid
order by oid;
```

## 4. 我的解答

```sql
with article as (
    select stack(
        6,
        1, 0,
        2, 0,
        3, 1,
        4, 1,
        5, 2,
        6, 0
    ) as (id, oid)
),
marked_data as (
    select
        id,
        oid,
        COLLECT_SET(if(oid = 0, id, null)) over() origin_id
    from
        article
)
select
    oid,
    sum(if(oid in origin_id, 1, 0)) sums
from
    marked_data
group by
    oid
having oid <> 0
order by oid;
```

## 5. 我的解答正确版本

```sql
with article as (
    select stack(
        6,
        1, 0,
        2, 0,
        3, 1,
        4, 1,
        5, 2,
        6, 0
    ) as (id, oid)
),
marked_data as (
    select
        id,
        oid,
        collect_set(if(oid = 0, id, null)) over () as origin_id
    from article
)
select
    oid,
    sum(if(array_contains(origin_id, oid), 1, 0)) as sums
from marked_data
where oid <> 0
group by oid
order by oid;
```

## 6. 我的复盘要点

- 【下次先想】这题不是普通自关联题，题目明确说不用 `join`，你的思路是先用窗口函数把所有原创文章 `id` 收集成一个集合状态，这个方向是对的。
- 【修正记法】`collect_set(if(oid = 0, id, null)) over()` 的作用是给每一行都挂上“原创文章 id 集合”，后面就能直接判断当前 `oid` 是否在这个集合里。
- 【本题坑点】Hive 里判断数组是否包含某个值要用 `array_contains(origin_id, oid)`；`oid in origin_id` 更像 SQL 的枚举或子查询判断，不适合这里的数组字段。
- 【本题坑点】最后要先 `where oid <> 0` 去掉原创文章本身，再按 `oid` 分组统计；`having oid <> 0` 也能表达分组后过滤，但这里放在聚合前更清楚。
