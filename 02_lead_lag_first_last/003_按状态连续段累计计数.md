# 按状态连续段累计计数

## 1. 题目描述

给定表 `temp`，包含字段 `A` 和 `B`。要求按 `A` 升序排列后，对 `B` 列进行处理，生成新列 `C`。

| 字段名 | 含义 |
|---|---|
| `A` | 排序字段，示例中为年份 |
| `B` | 状态值 |

示例数据：

| A | B |
|---:|---:|
| 2010 | 1 |
| 2011 | 1 |
| 2012 | 1 |
| 2013 | 0 |
| 2014 | 0 |
| 2015 | 1 |
| 2016 | 1 |
| 2017 | 1 |
| 2018 | 0 |
| 2019 | 0 |

要求生成 `C` 列：如果 `B` 列值不变，则 `C` 在当前连续段内累计计数；如果 `B` 列值发生变化，则 `C` 从 1 重新开始计数。

> 本题口径：连续段只按 `A` 排序后的相邻行判断，`B` 相同但中间被其他值打断后，后面再次出现相同 `B` 时要重新开始计数。

## 2. 期望结果

| A | B | C |
|---:|---:|---:|
| 2010 | 1 | 1 |
| 2011 | 1 | 2 |
| 2012 | 1 | 3 |
| 2013 | 0 | 1 |
| 2014 | 0 | 2 |
| 2015 | 1 | 1 |
| 2016 | 1 | 2 |
| 2017 | 1 | 3 |
| 2018 | 0 | 1 |
| 2019 | 0 | 2 |

## 3. 标准答案

```sql
with data as (
    select 2010 as A, 1 as B union all
    select 2011 as A, 1 as B union all
    select 2012 as A, 1 as B union all
    select 2013 as A, 0 as B union all
    select 2014 as A, 0 as B union all
    select 2015 as A, 1 as B union all
    select 2016 as A, 1 as B union all
    select 2017 as A, 1 as B union all
    select 2018 as A, 0 as B union all
    select 2019 as A, 0 as B
),
ordered_data as (
    select
        A,
        B,
        lag(B) over(order by A) as prev_B
    from data
),
marked_data as (
    select
        A,
        B,
        case
            when prev_B is null then 1
            when prev_B <> B then 1
            else 0
        end as is_new_group
    from ordered_data
),
grouped_data as (
    select
        A,
        B,
        sum(is_new_group) over(
            order by A
            rows between unbounded preceding and current row
        ) as group_id
    from marked_data
)
select
    A,
    B,
    row_number() over(partition by group_id order by A) as C
from grouped_data
order by A;
```

## 4. 我的解答

```sql
WITH data AS (
    -- 模拟原始数据
    SELECT 2010 AS A, 1 AS B UNION ALL
    SELECT 2011 AS A, 1 AS B UNION ALL
    SELECT 2012 AS A, 1 AS B UNION ALL
    SELECT 2013 AS A, 0 AS B UNION ALL
    SELECT 2014 AS A, 0 AS B UNION ALL
    SELECT 2015 AS A, 1 AS B UNION ALL
    SELECT 2016 AS A, 1 AS B UNION ALL
    SELECT 2017 AS A, 1 AS B UNION ALL
    SELECT 2018 AS A, 0 AS B UNION ALL
    SELECT 2019 AS A, 0 AS B
),
ordered_data as (
    select 
        A,
        B,
        lag(B) over(order by A) lag_B
    from 
        data
),
marked_data as (
    select 
        A,
        B,
        sum(
            case    
                when lag_B <> B then 1 
                when lag_B is null then 1
                else 0 
            end 
        ) over(order by A) mark_B
    from 
        ordered_data
)
select 
    A,
    B,
    row_number() over(partition by mark_B order by A)
from 
    marked_data
```

## 5. 我的解答正确版本

```sql
with data as (
    select 2010 as A, 1 as B union all
    select 2011 as A, 1 as B union all
    select 2012 as A, 1 as B union all
    select 2013 as A, 0 as B union all
    select 2014 as A, 0 as B union all
    select 2015 as A, 1 as B union all
    select 2016 as A, 1 as B union all
    select 2017 as A, 1 as B union all
    select 2018 as A, 0 as B union all
    select 2019 as A, 0 as B
),
ordered_data as (
    select 
        A,
        B,
        lag(B) over(order by A) as lag_B
    from data
),
marked_data as (
    select 
        A,
        B,
        sum(
            case    
                when lag_B is null then 1
                when lag_B <> B then 1 
                else 0 
            end 
        ) over(
            order by A
            rows between unbounded preceding and current row
        ) as mark_B
    from ordered_data
)
select 
    A,
    B,
    row_number() over(partition by mark_B order by A) as C
from marked_data
order by A;
```

## 6. 我的复盘要点

- 【本题坑点】不能直接按 `B` 分组计数，因为 `B = 1` 出现了两段，中间被 `B = 0` 打断后要重新从 1 开始。
- 【修正记法】先用 `lag(B)` 判断当前行是否开启新段，再对新段标记累计求和，得到连续段编号。
- 【下次先想】状态连续段题不要先想 `group by 状态值`，要先想“相邻行有没有变化”。
- 【本题坑点】最后的 `C` 是连续段内部行号，所以应该按连续段编号分区，而不是按原始状态值 `B` 分区。
