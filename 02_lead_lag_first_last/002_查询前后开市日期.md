# 查询前后开市日期

## 1. 题目描述

给定表 `data`，每一行表示某个 `id` 在某一天是否开市。

| 字段名 | 含义 |
|---|---|
| `id` | 对象 ID |
| `date` | 日期 |
| `is_open` | 是否开市，`1` 表示开市，`0` 表示不开市 |

示例数据：

| id | date | is_open |
|---:|---|---:|
| 1 | 2023/1/2 | 1 |
| 1 | 2023/1/3 | 1 |
| 1 | 2023/1/4 | 0 |
| 1 | 2023/1/5 | 0 |
| 1 | 2023/1/6 | 1 |
| 1 | 2023/1/7 | 0 |
| 1 | 2023/1/8 | 1 |
| 1 | 2023/1/9 | 1 |
| 1 | 2023/1/10 | 0 |
| 1 | 2023/1/11 | 0 |
| 1 | 2023/1/12 | 0 |
| 1 | 2023/1/13 | 0 |

要求将表 A 转换成表 B：在原字段基础上，补充当前日期之前最近的开市日期 `pre_date`，以及当前日期之后最近的开市日期 `next_date`。

> 本题口径：`pre_date` 和 `next_date` 都不包含当前行本身。如果当前行之前没有开市日，则 `pre_date` 填 `1900/1/1`；如果当前行之后没有开市日，则 `next_date` 填 `9999/1/1`。

## 2. 期望结果

| id | date | is_open | pre_date | next_date |
|---:|---|---:|---|---|
| 1 | 2023/1/2 | 1 | 1900/1/1 | 2023/1/3 |
| 1 | 2023/1/3 | 1 | 2023/1/2 | 2023/1/6 |
| 1 | 2023/1/4 | 0 | 2023/1/3 | 2023/1/6 |
| 1 | 2023/1/5 | 0 | 2023/1/3 | 2023/1/6 |
| 1 | 2023/1/6 | 1 | 2023/1/3 | 2023/1/8 |
| 1 | 2023/1/7 | 0 | 2023/1/6 | 2023/1/8 |
| 1 | 2023/1/8 | 1 | 2023/1/6 | 2023/1/9 |
| 1 | 2023/1/9 | 1 | 2023/1/8 | 9999/1/1 |
| 1 | 2023/1/10 | 0 | 2023/1/9 | 9999/1/1 |
| 1 | 2023/1/11 | 0 | 2023/1/9 | 9999/1/1 |
| 1 | 2023/1/12 | 0 | 2023/1/9 | 9999/1/1 |
| 1 | 2023/1/13 | 0 | 2023/1/9 | 9999/1/1 |

## 3. 标准答案

```sql
with data as (
    select 1 as id, '2023-01-02' as dt, 1 as is_open union all
    select 1 as id, '2023-01-03' as dt, 1 as is_open union all
    select 1 as id, '2023-01-04' as dt, 0 as is_open union all
    select 1 as id, '2023-01-05' as dt, 0 as is_open union all
    select 1 as id, '2023-01-06' as dt, 1 as is_open union all
    select 1 as id, '2023-01-07' as dt, 0 as is_open union all
    select 1 as id, '2023-01-08' as dt, 1 as is_open union all
    select 1 as id, '2023-01-09' as dt, 1 as is_open union all
    select 1 as id, '2023-01-10' as dt, 0 as is_open union all
    select 1 as id, '2023-01-11' as dt, 0 as is_open union all
    select 1 as id, '2023-01-12' as dt, 0 as is_open union all
    select 1 as id, '2023-01-13' as dt, 0 as is_open
),
base as (
    select
        id,
        to_date(dt) as dt,
        is_open
    from data
),
filled_data as (
    select
        id,
        dt,
        is_open,
        coalesce(
            max(case when is_open = 1 then dt end) over(
                partition by id
                order by dt
                rows between unbounded preceding and 1 preceding
            ),
            to_date('1900-01-01')
        ) as pre_date,
        coalesce(
            min(case when is_open = 1 then dt end) over(
                partition by id
                order by dt
                rows between 1 following and unbounded following
            ),
            to_date('9999-01-01')
        ) as next_date
    from base
)
select
    id,
    date_format(dt, 'yyyy/M/d') as `date`,
    is_open,
    date_format(pre_date, 'yyyy/M/d') as pre_date,
    date_format(next_date, 'yyyy/M/d') as next_date
from filled_data
order by id, dt;
```

## 6. 我的复盘要点

- 【我的卡点】`pre_date` 和 `next_date` 都不包含当前行，所以向前找要截止到 `1 preceding`，向后找要从 `1 following` 开始。
- 【修正记法】`first_value(expr, true)` / `last_value(expr, true)` 的第二个参数 `true` 表示忽略空值，可以跳过 `is_open = 0` 的日期。
- 【错误原因】原 SQL 里生成的是 `preve_dt`，最终查询引用 `prev_dt`，这个别名不一致会直接导致字段找不到。
- 【本题坑点】默认前值应按题目补 `1900-01-01`，不是 `1999-01-01`；日期展示格式用 `yyyy/M/d` 更贴近期望结果。
- 【下次先想】找“之前最近”可以用前向窗口，找“之后最近”可以用后向窗口，关键是先确认是否包含当前行。
