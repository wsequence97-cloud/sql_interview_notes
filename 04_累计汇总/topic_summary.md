# 累计汇总

## 1. 题型定义

累计汇总类题目主要考察累计和、滚动窗口、移动平均、截止当前行的聚合指标等。

## 2. 核心套路

* 按时间排序。
* 使用 `sum() over()` 做累计。
* 使用窗口边界控制滚动范围。
* 先按天或业务粒度聚合，再做窗口计算。

## 3. 常用 Hive 函数

* `sum() over()`
* `avg() over()`
* `count() over()`
* `rows between ...`
* `date_sub()`
* `date_add()`

## 4. 通用解题模板

```sql
with data as (
    select '2023-01-01' as dt, 100 as amount union all
    select '2023-01-02' as dt, 200 as amount
),
daily_data as (
    select
        to_date(dt) as dt,
        sum(amount) as amount
    from data
    group by to_date(dt)
)
select
    dt,
    amount,
    sum(amount) over(order by dt rows between unbounded preceding and current row) as cum_amount
from daily_data;
```

## 6. 本题型高频易错点

* 没有先按统计粒度聚合。
* 窗口边界写错，导致累计和滚动窗口混淆。
* 日期缺失时没有补齐日期，导致滚动窗口口径偏差。
* 同一统计周期内有多条明细时，先判断题目是否要求去重；如果不去重，就应先 `sum()` 到周期粒度。
* 日期字符串如果只需要月份，可以用 `from_unixtime(unix_timestamp(日期字段, '输入格式'), 'yyyy-MM')` 直接格式化。
* 同时在线类题目可以把开始时间打 `+1`、结束时间打 `-1`，再按时间累计并取最大值。
* 如果开始和结束时间相同，要先确认同一时刻是先加还是先减，否则最大在线人数可能差 1。
* “每小时有多少人在线”通常要把在线区间展开到小时粒度；只按登录小时和退出小时打 `+1/-1`，得到的是每小时净变化，不是小时覆盖人数。
* “近 N 天累计”要区分日期范围窗口和行数窗口：`range` 看时间范围，`rows` 看前 N 行。
* “当前是否历史新高/新低”类题目通常要同时计算历史累计极值，并筛出最新一条记录再判断。

## 7. 面试表达模板

这类题我会先把原始数据聚合到题目要求的统计粒度，然后按时间排序，用窗口聚合函数配合窗口边界计算累计或滚动指标。
