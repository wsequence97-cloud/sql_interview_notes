# 统计每小时同时在线人数

## 1. 题目描述

给定直播间用户在线区间表 `data`，字段如下：

| 字段名 | 含义 |
|---|---|
| `room_id` | 直播间 id |
| `user_id` | 用户 id |
| `login_time` | 用户进入直播间时间 |
| `logout_time` | 用户退出直播间时间 |

示例数据：

| room_id | user_id | login_time | logout_time |
|---|---:|---|---|
| a | 1 | 2020-11-25 12:00:00 | 2020-11-25 13:15:00 |
| a | 1 | 2020-11-25 12:30:00 | 2020-11-25 12:40:00 |
| a | 2 | 2020-11-25 12:30:30 | 2020-11-25 14:35:00 |
| a | 3 | 2020-11-25 12:35:30 | 2020-11-25 15:40:00 |
| a | 4 | 2020-11-25 12:38:30 | 2020-11-25 12:40:00 |

要求查询：统计某个时间段内，每个直播间每小时内的同时在线人数。

> 本题口径：只要用户在线区间覆盖某个小时，就把该用户计入该小时；同一个用户在同一个小时内出现多条记录时，按 `count(distinct user_id)` 去重。

## 2. 期望结果

| room_id | on_time | on_time_cnt |
|---|---:|---:|
| a | 12 | 4 |
| a | 13 | 3 |
| a | 14 | 2 |
| a | 15 | 1 |

## 3. 标准答案

```sql
with data as (
    select 'a' as room_id, 1 as user_id, '2020-11-25 12:00:00' as login_time, '2020-11-25 13:15:00' as logout_time union all
    select 'a' as room_id, 1 as user_id, '2020-11-25 12:30:00' as login_time, '2020-11-25 12:40:00' as logout_time union all
    select 'a' as room_id, 2 as user_id, '2020-11-25 12:30:30' as login_time, '2020-11-25 14:35:00' as logout_time union all
    select 'a' as room_id, 3 as user_id, '2020-11-25 12:35:30' as login_time, '2020-11-25 15:40:00' as logout_time union all
    select 'a' as room_id, 4 as user_id, '2020-11-25 12:38:30' as login_time, '2020-11-25 12:40:00' as logout_time
),
temp_data as (
    select
        room_id,
        user_id,
        hour(login_time) as min_time,
        hour(logout_time) as max_time,
        split(space(hour(logout_time) - hour(login_time)), ' ') as dis
    from data
)
select
    room_id,
    on_time,
    count(distinct user_id) as on_time_cnt
from (
    select
        room_id,
        user_id,
        min_time + dis_index as on_time
    from temp_data
    lateral view posexplode(dis) n as dis_index, dis_data
) t
group by
    room_id,
    on_time
order by
    room_id,
    on_time;
```

        room_id,
        user_id,

        1 as user_type

        -1 as user_type

## 5. 我的解答正确版本

```sql
with data as (
    select 'a' as room_id, 1 as user_id, '2020-11-25 12:00:00' as login_time, '2020-11-25 13:15:00' as logout_time union all
    select 'a' as room_id, 1 as user_id, '2020-11-25 12:30:00' as login_time, '2020-11-25 12:40:00' as logout_time union all
    select 'a' as room_id, 2 as user_id, '2020-11-25 12:30:30' as login_time, '2020-11-25 14:35:00' as logout_time union all
    select 'a' as room_id, 3 as user_id, '2020-11-25 12:35:30' as login_time, '2020-11-25 15:40:00' as logout_time union all
    select 'a' as room_id, 4 as user_id, '2020-11-25 12:38:30' as login_time, '2020-11-25 12:40:00' as logout_time
),
base as (
    select
        room_id,
        user_id,
        hour(login_time) as start_hour,
        hour(logout_time) as end_hour
    from data
),
expanded_data as (
    select
        room_id,
        user_id,
        start_hour + hour_index as on_time
    from (
        select
            room_id,
            user_id,
            start_hour,
            split(space(end_hour - start_hour), ' ') as hour_arr
        from base
    ) t
    lateral view posexplode(hour_arr) p as hour_index, hour_value
)
select
    room_id,
    on_time,
    count(distinct user_id) as on_time_cnt
from expanded_data
group by
    room_id,
    on_time
order by
    room_id,
    on_time;
```

## 6. 我的复盘要点

- 【我的卡点】我没想出来“用户在哪些整点在线”要先把在线区间展开成小时序列；例如 `12:00` 到 `13:15` 覆盖的是 12 点和 13 点，而不是只在登录小时加、退出小时减。
- 【错误原因】我原来把登录小时记 `+1`、退出小时记 `-1` 后按小时求和，这只能得到“该小时净变化”，不是“该小时覆盖在线人数”，会漏掉跨小时仍在线的用户。
- 【修正记法】`split(space(hour(logout_time) - hour(login_time)), ' ')` 是为了造一个长度为“小时差 + 1”的数组；后面用数组下标 `0,1,2...` 加上起始小时，生成用户覆盖到的每个整点小时。
- 【我的卡点】`lateral view` 是 Hive 用来把原表行和展开结果拼在一起的语法；`explode` 只能拆数组元素，`posexplode` 会同时返回数组下标和元素，所以本题必须用 `posexplode` 拿到 `dis_index`。
- 【本题坑点】`user_id = 1` 在 12 点有重叠区间，按用户人数口径必须 `count(distinct user_id)`，否则 12 点会把同一个用户重复计算。
