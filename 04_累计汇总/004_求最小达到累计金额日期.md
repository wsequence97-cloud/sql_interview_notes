# 求最小达到累计金额日期

## 1. 题目描述

给定订单消费表 `test`，每一行表示某个用户在某一天的一笔订单消费。

| 字段名 | 含义 |
|---|---|
| `user_id` | 用户 ID |
| `dt` | 订单日期 |
| `price` | 订单金额 |

示例数据：

| user_id | dt | price |
|---:|---|---:|
| 1001 | 2025-03-01 | 3000 |
| 1002 | 2025-03-15 | 6000 |
| 1001 | 2025-03-05 | 4000 |
| 1003 | 2025-03-03 | 1000 |
| 1002 | 2025-03-06 | 3000 |
| 1001 | 2025-03-10 | 4000 |
| 1003 | 2025-03-01 | 1000 |
| 1002 | 2025-03-02 | 2000 |
| 1003 | 2025-03-20 | 8500 |

要求查询：计算每个消费者历史上首次在近 30 天周期内累计消费金额达到 1W 的日期。

> 本题口径：按用户维度计算；“达到 1W”按 `>= 10000` 理解。

## 2. 期望结果

| user_id | first_dt |
|---:|---|
| 1001 | 2025-03-10 |
| 1002 | 2025-03-15 |
| 1003 | 2025-03-20 |

## 3. 标准答案

```sql
with data as (
    select 1001 as user_id, '2025-03-01' as dt, 3000 as price union all
    select 1002 as user_id, '2025-03-15' as dt, 6000 as price union all
    select 1001 as user_id, '2025-03-05' as dt, 4000 as price union all
    select 1003 as user_id, '2025-03-03' as dt, 1000 as price union all
    select 1002 as user_id, '2025-03-06' as dt, 3000 as price union all
    select 1001 as user_id, '2025-03-10' as dt, 4000 as price union all
    select 1003 as user_id, '2025-03-01' as dt, 1000 as price union all
    select 1002 as user_id, '2025-03-02' as dt, 2000 as price union all
    select 1003 as user_id, '2025-03-20' as dt, 8500 as price
),
base as (
    select
        user_id,
        to_date(dt) as dt,
        datediff(to_date(dt), to_date('1970-01-01')) as dt_day,
        price
    from data
),
rolling_data as (
    select
        user_id,
        dt,
        sum(price) over(
            partition by user_id
            order by dt_day
            range between 30 preceding and current row
        ) as rolling_price
    from base
)
select
    user_id,
    date_format(min(dt), 'yyyy-MM-dd') as first_dt
from rolling_data
where rolling_price >= 10000
group by user_id
order by user_id;
```

## 4. 我的解答

```sql
WITH data AS (
-- 造数：用户订单表（user_id, 订单日期dt, 金额price）
SELECT 1001 AS user_id, '2025-03-01' AS dt, 3000 AS price UNION ALL
SELECT 1002 AS user_id, '2025-03-15' AS dt, 6000 AS price UNION ALL -- 首次达标
SELECT 1001 AS user_id, '2025-03-05' AS dt, 4000 AS price UNION ALL
SELECT 1003 AS user_id, '2025-03-03' AS dt, 1000 AS price UNION ALL
SELECT 1002 AS user_id, '2025-03-06' AS dt, 3000 AS price UNION ALL
SELECT 1001 AS user_id, '2025-03-10' AS dt, 4000 AS price UNION ALL  -- 近30天累计 11000 ✅ 首次达标
SELECT 1003 AS user_id, '2025-03-01' AS dt, 1000 AS price UNION ALL
SELECT 1002 AS user_id, '2025-03-02' AS dt, 2000 AS price UNION ALL
SELECT 1003 AS user_id, '2025-03-20' AS dt, 8500 AS price           -- 首次达标
),
base as (
    select 
        user_id,
        TO_DATE(dt) dt,
        price 
    from 
        data
),
sum_data as (
    select 
        user_id,
        dt,
        sum(price) over(partition by user_id order by dt rows between 30 preceding and current row) cnt_price
    from 
        base 
)
select 
    user_id,
    dt,
    min(cnt_price)
from 
    sum_data
group by 
    user_id,dt
having min(cnt_price) >= 10000
```

## 6. 我的复盘要点

- 【我的卡点】这题核心不是普通累计和，而是“每个日期往前看近 30 天”的滚动累计；要先想清楚窗口是按日期范围滚动。
- 【错误原因】我原 SQL 用的是 `rows between 30 preceding and current row`，它表示当前行往前 30 行，不是真正的近 30 天；如果日期缺失或一天多单，口径会偏。
- 【修正记法】日期范围窗口可以先把 `dt` 转成数值天数 `dt_day`，再用 `range between 30 preceding and current row` 表示当前日期往前 30 天范围。
- 【错误原因】最后不能按 `user_id, dt` 分组后直接 `having min(cnt_price) >= 10000`，这样会输出每个达标日期；题目要“首次达标”，应先筛达标行，再按 `user_id` 取 `min(dt)`。
- 【本题坑点】题目文字说“达到 1W”通常按 `>= 10000` 理解；如果面试题给的是 `> 10000`，要先和题面或面试官确认阈值口径。
