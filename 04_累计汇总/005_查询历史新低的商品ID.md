# 查询历史新低的商品 ID

## 1. 题目描述

给定商品价格变化表 `data_tbl`，每一行表示某个商品在某个时间点的一次价格变更。

| 字段名 | 含义 |
|---|---|
| `id` | 商品 ID |
| `start_price` | 商品变更前价格 |
| `after_price` | 商品变更后价格 |
| `ts` | 价格变更时间戳 |

示例数据：

| id | start_price | after_price | ts |
|---|---:|---:|---:|
| a111 | 4 | 9 | 20230110000 |
| a111 | 9 | 8 | 20230110100 |
| a111 | 8 | 6 | 20230110200 |
| a112 | 5 | 4 | 20230110000 |
| a112 | 4 | 2 | 20230110100 |

要求查询：当前价格处于历史新低的商品 ID。

> 本题口径：每个商品按 `ts` 判断最新一条价格变更记录；如果最新记录的 `after_price` 小于等于该商品历史上截至当前记录之前价格中的最低值，则认为当前价格处于历史新低。

## 2. 期望结果

| id |
|---|
| a112 |

## 3. 标准答案

```sql
with data_tbl as (
    select 'a111' as id, 4 as start_price, 9 as after_price, 20230110000 as ts union all
    select 'a111' as id, 9 as start_price, 8 as after_price, 20230110100 as ts union all
    select 'a111' as id, 8 as start_price, 6 as after_price, 20230110200 as ts union all
    select 'a112' as id, 5 as start_price, 4 as after_price, 20230110000 as ts union all
    select 'a112' as id, 4 as start_price, 2 as after_price, 20230110100 as ts
),
price_marked as (
    select
        id,
        start_price,
        after_price,
        ts,
        min(start_price) over(
            partition by id
            order by ts
            rows between unbounded preceding and current row
        ) as history_lowest_price,
        row_number() over(
            partition by id
            order by ts desc
        ) as rn
    from data_tbl
)
select
    id
from price_marked
where rn = 1
  and after_price <= history_lowest_price
order by id;
```

## 4. 我的解答

```sql
WITH DATA  AS (  -- 定义 CTE，命名为 price_changes
    SELECT 'a111' AS id, 4 AS start_price, 9 AS after_price, '20230110000' AS ts UNION ALL
    SELECT 'a111', 9, 8, '20230110100' UNION ALL
    SELECT 'a111', 8, 6, '20230110200' UNION ALL
    SELECT 'a112', 5, 4, '20230110000' UNION ALL
    SELECT 'a112', 4, 2, '20230110100'
),
low_data as (
    select 
        id,
        ts,
        min(strat_price) over(partition by id order by ts) min_start_price,
        after_price,
        row_number() over(partition by id order by ts desc) rank_ts
    from 
        data 
)
select 
    id
from 
    low_data
where 
    after_price < min_start_price
and 
    rank = 1
```

## 5. 我的解答正确版本

```sql
with data as (
    select 'a111' as id, 4 as start_price, 9 as after_price, '20230110000' as ts union all
    select 'a111' as id, 9 as start_price, 8 as after_price, '20230110100' as ts union all
    select 'a111' as id, 8 as start_price, 6 as after_price, '20230110200' as ts union all
    select 'a112' as id, 5 as start_price, 4 as after_price, '20230110000' as ts union all
    select 'a112' as id, 4 as start_price, 2 as after_price, '20230110100' as ts
),
low_data as (
    select 
        id,
        ts,
        min(start_price) over(
            partition by id
            order by ts
            rows between unbounded preceding and current row
        ) as min_start_price,
        after_price,
        row_number() over(partition by id order by ts desc) as rank_ts
    from data
)
select 
    id
from low_data
where after_price <= min_start_price
  and rank_ts = 1
order by id;
```

## 6. 我的复盘要点

- 【修正记法】这题分两步：先按商品和时间累计历史最低 `start_price`，再只取每个商品最新一条记录判断 `after_price` 是否创新低。
- 【本题坑点】不能直接拿所有行的 `after_price` 判断，否则会把历史某次降价也算进去；题目问的是“当前价格”是否处于历史新低。
- 【下次先想】遇到“历史最高/历史最低 + 当前状态”类题，要同时准备两个窗口：一个累计极值窗口，一个最新行 `row_number()`。
- 【错误原因】我原 SQL 里把 `start_price` 写成了 `strat_price`，会直接导致字段不存在；最终筛选也应该用生成的 `rank_ts`，不能写成 `rank = 1`。
- 【本题坑点】截图答案用的是 `after_price <= min_start_price`，表示等于历史最低也算处于历史新低；如果题目要求“严格新低”，才改成 `<`。
