# 核心营位帕累托分析

## 1. 题目描述

- 牛客题号：SQL151
- 难度：较难
- 题目来源：[25年携程-露营地旺季「核心营位」帕累托分析](https://www.nowcoder.com/practice/77f924a37530406ca86455381b6d1a5a)
- 题解依据：[题解｜25年携程-露营地旺季「核心营位」帕累托分析](https://www.nowcoder.com/discuss/892386965419487232)

某网红连锁露营地在 2025 年旺季结束后进行经营复盘，需要按照实际入账金额识别贡献累计营收前 80% 的核心营位。

给定营位档案表 `t_camp_site`，字段如下：

| 字段名 | 类型 | 含义 |
|---|---|---|
| `site_id` | BIGINT | 营位编号，主键 |
| `site_name` | VARCHAR(64) | 营位名称 |
| `site_type` | VARCHAR(16) | 住宿类型，取值为 `tent`、`cabin`、`rv` 或 `canopy` |
| `capacity` | TINYINT | 最大容纳人数 |
| `daily_price` | DECIMAL(7,2) | 每晚定价 |
| `region` | VARCHAR(32) | 所在片区 |

只考虑 `check_in` 在 2025-04-01 至 2025-09-30 之间的预订。按营位统计旺季预订单数、总入住晚数和实际入账金额，再按 `actual_revenue` 降序、`site_id` 升序计算累计入账金额及累计占比，输出所有核心营位。

实际入账比例为：`sunny` 和 `cloudy` 按 100%，`light_rain` 按 70%，`heavy_rain` 按 50%，`storm` 按 0%。

> 本题口径：一个营位属于核心营位，当且仅当累加到本行之前的累计入账金额仍小于总入账金额的 80%；因此首次达到或超过 80% 的营位也必须包含。总入住晚数按每单 `check_out - check_in` 的天数求和，金额和累计占比均四舍五入保留 2 位小数。

## 3. 标准答案

以下答案沿用公开题解的营位聚合、累计窗口与阈值判断思路，并按照原题天气规则改写为可直接运行的 Hive SQL。

```sql
WITH data AS (
    SELECT stack(
        8,
        1, '湖光晨雾', 'tent',   4, CAST(180.00 AS DECIMAL(7, 2)), '湖景区',
        2, '林间小屋A', 'cabin', 6, CAST(480.00 AS DECIMAL(7, 2)), '森林区',
        3, '云顶天幕', 'canopy', 8, CAST(280.00 AS DECIMAL(7, 2)), '草甸区',
        4, '星辰房车', 'rv',     4, CAST(680.00 AS DECIMAL(7, 2)), '湖景区',
        5, '银杏木屋', 'cabin',  5, CAST(580.00 AS DECIMAL(7, 2)), '银杏林',
        6, '溪畔帐篷', 'tent',   3, CAST(220.00 AS DECIMAL(7, 2)), '溪谷区',
        7, '山脊木屋', 'cabin',  6, CAST(520.00 AS DECIMAL(7, 2)), '高山区',
        8, '林深房车', 'rv',     4, CAST(750.00 AS DECIMAL(7, 2)), '森林区'
    ) AS (site_id, site_name, site_type, capacity, daily_price, region)
),
t_booking AS (
    SELECT stack(
        16,
        401, 1, '13800000001', '2025-05-01', '2025-05-04', 3, 'sunny',      CAST(1800.00 AS DECIMAL(8, 2)),
        402, 2, '13800000002', '2025-05-10', '2025-05-12', 5, 'sunny',      CAST(1500.00 AS DECIMAL(8, 2)),
        403, 2, '13800000003', '2025-06-15', '2025-06-16', 4, 'sunny',      CAST(1000.00 AS DECIMAL(8, 2)),
        404, 3, '13800000004', '2025-07-08', '2025-07-12', 7, 'sunny',      CAST(2000.00 AS DECIMAL(8, 2)),
        405, 4, '13800000005', '2025-05-20', '2025-05-23', 4, 'sunny',      CAST(3000.00 AS DECIMAL(8, 2)),
        406, 4, '13800000006', '2025-08-15', '2025-08-17', 3, 'sunny',      CAST(2000.00 AS DECIMAL(8, 2)),
        407, 5, '13800000007', '2025-04-12', '2025-04-16', 5, 'sunny',      CAST(4000.00 AS DECIMAL(8, 2)),
        408, 5, '13800000008', '2025-06-20', '2025-06-23', 5, 'sunny',      CAST(2500.00 AS DECIMAL(8, 2)),
        409, 5, '13800000009', '2025-09-05', '2025-09-07', 4, 'sunny',      CAST(1500.00 AS DECIMAL(8, 2)),
        410, 6, '13800000010', '2025-06-10', '2025-06-12', 3, 'sunny',      CAST(1200.00 AS DECIMAL(8, 2)),
        411, 7, '13800000011', '2025-07-22', '2025-07-24', 6, 'sunny',      CAST(2000.00 AS DECIMAL(8, 2)),
        412, 7, '13800000012', '2025-08-10', '2025-08-13', 5, 'heavy_rain', CAST(3000.00 AS DECIMAL(8, 2)),
        413, 8, '13800000013', '2025-05-15', '2025-05-18', 4, 'sunny',      CAST(3000.00 AS DECIMAL(8, 2)),
        414, 8, '13800000014', '2025-07-04', '2025-07-06', 4, 'sunny',      CAST(2000.00 AS DECIMAL(8, 2)),
        415, 8, '13800000015', '2025-09-12', '2025-09-13', 3, 'sunny',      CAST(1000.00 AS DECIMAL(8, 2)),
        416, 8, '13800000016', '2025-11-15', '2025-11-17', 4, 'sunny',      CAST(5000.00 AS DECIMAL(8, 2))
    ) AS (
        booking_id,
        site_id,
        guest_phone,
        check_in,
        check_out,
        party_size,
        weather,
        paid_amount
    )
),
t_camp_site AS (
    SELECT
        CAST(site_id AS BIGINT) AS site_id,
        site_name,
        site_type,
        CAST(capacity AS TINYINT) AS capacity,
        daily_price,
        region
    FROM data
),
bookings AS (
    SELECT
        CAST(booking_id AS BIGINT) AS booking_id,
        CAST(site_id AS BIGINT) AS site_id,
        guest_phone,
        to_date(check_in) AS check_in,
        to_date(check_out) AS check_out,
        CAST(party_size AS TINYINT) AS party_size,
        weather,
        paid_amount
    FROM t_booking
),
site_metrics AS (
    SELECT
        s.site_id,
        s.site_name,
        s.site_type,
        s.region,
        count(*) AS booking_cnt,
        sum(datediff(b.check_out, b.check_in)) AS total_nights,
        CAST(
            round(
                sum(
                    CASE
                        WHEN b.weather IN ('sunny', 'cloudy') THEN b.paid_amount
                        WHEN b.weather = 'light_rain' THEN b.paid_amount * CAST(0.70 AS DECIMAL(3, 2))
                        WHEN b.weather = 'heavy_rain' THEN b.paid_amount * CAST(0.50 AS DECIMAL(3, 2))
                        WHEN b.weather = 'storm' THEN CAST(0.00 AS DECIMAL(8, 2))
                        ELSE CAST(0.00 AS DECIMAL(8, 2))
                    END
                ),
                2
            ) AS DECIMAL(20, 2)
        ) AS actual_revenue
    FROM t_camp_site s
    INNER JOIN bookings b
        ON s.site_id = b.site_id
    WHERE b.check_in BETWEEN to_date('2025-04-01') AND to_date('2025-09-30')
    GROUP BY s.site_id, s.site_name, s.site_type, s.region
),
pareto_totals AS (
    SELECT
        site_id,
        site_name,
        site_type,
        region,
        booking_cnt,
        total_nights,
        actual_revenue,
        sum(actual_revenue) OVER (
            ORDER BY actual_revenue DESC, site_id ASC
            ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
        ) AS cumulative_revenue,
        sum(actual_revenue) OVER () AS total_revenue
    FROM site_metrics
),
pareto_with_cutoff AS (
    SELECT
        site_id,
        site_name,
        site_type,
        region,
        booking_cnt,
        total_nights,
        actual_revenue,
        CAST(round(cumulative_revenue, 2) AS DECIMAL(20, 2)) AS cumulative_revenue,
        CAST(
            round(cumulative_revenue * 100.0 / total_revenue, 2)
            AS DECIMAL(10, 2)
        ) AS cumulative_pct,
        total_revenue,
        lag(cumulative_revenue) OVER (
            ORDER BY actual_revenue DESC, site_id ASC
        ) AS previous_cumulative_revenue
    FROM pareto_totals
)
SELECT
    site_id,
    site_name,
    site_type,
    region,
    booking_cnt,
    total_nights,
    actual_revenue,
    cumulative_revenue,
    cumulative_pct
FROM pareto_with_cutoff
WHERE coalesce(previous_cumulative_revenue, CAST(0.00 AS DECIMAL(20, 2)))
    < total_revenue * CAST(0.80 AS DECIMAL(3, 2))
ORDER BY actual_revenue DESC, site_id ASC;
```
