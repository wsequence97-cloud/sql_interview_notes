# 近7天骑手履约时效看板

## 1. 题目描述

- 牛客题号：SQL129
- 难度：中等
- 题目来源：[近7天骑手履约时效看板](https://www.nowcoder.com/practice/25af5a3296c747f5b01fc589f1568514)
- 题解依据：[题解｜近7天骑手履约时效看板](https://www.nowcoder.com/discuss/873357140038082560)

出行平台需要制作“近7天骑手履约时效看板”，按城市评估每位骑手近 7 天的送达单量、准时单量、准时率和平均配送时长，并在城市内排名。

给定骑手信息表 `courier`，字段如下：

| 字段名 | 类型 | 含义 |
|---|---|---|
| `courier_id` | INT | 骑手 ID，主键 |
| `courier_name` | VARCHAR(64) | 骑手姓名，非空 |
| `city` | VARCHAR(32) | 所在城市，非空 |
| `hired_at` | DATE | 入职日期，非空 |

示例数据：

| courier_id | courier_name | city | hired_at |
|---:|---|---|---|
| 1 | Andy | Beijing | 2024-01-01 |
| 2 | Ben | Beijing | 2024-02-01 |
| 3 | Chen | Shanghai | 2024-03-01 |

给定包裹履约表 `parcel`，字段如下：

| 字段名 | 类型 | 含义 |
|---|---|---|
| `parcel_id` | BIGINT | 包裹 ID，主键 |
| `courier_id` | INT | 骑手 ID，非空 |
| `shipped_ts` | DATETIME | 骑手取件时间，非空 |
| `delivered_ts` | DATETIME | 实际送达时间，非空，仅统计已送达记录 |
| `promised_minutes` | INT | 承诺时效，单位为分钟，非空 |

示例数据：

| parcel_id | courier_id | shipped_ts | delivered_ts | promised_minutes |
|---:|---:|---|---|---:|
| 90001 | 1 | 2024-08-01 10:00:00 | 2024-08-01 10:40:00 | 45 |
| 90002 | 1 | 2024-08-03 09:00:00 | 2024-08-03 10:10:00 | 60 |
| 90003 | 2 | 2024-08-02 09:00:00 | 2024-08-02 09:30:00 | 40 |
| 90004 | 2 | 2024-08-05 08:30:00 | 2024-08-05 09:15:00 | 45 |
| 90005 | 3 | 2024-08-04 11:00:00 | 2024-08-04 11:50:00 | 60 |
| 80001 | 1 | 2024-07-20 09:00:00 | 2024-07-20 10:00:00 | 45 |

要求以全部已送达记录中最新的 `delivered_ts` 所在日期为基准日，对每位骑手统计：

- `orders_7d`：近 7 天送达单量。
- `on_time_7d`：近 7 天准时单量，实际送达时长小于等于承诺时效即为准时。
- `on_time_rate`：准时单量除以送达单量，四舍五入保留 2 位小数；送达单量为 0 时记为 0。
- `avg_minutes_7d`：平均送达分钟数，四舍五入保留 2 位小数。
- `rank_in_city`：同城市内按 `on_time_rate` 降序、`avg_minutes_7d` 升序使用 `RANK()` 排名。

只输出 `orders_7d > 0` 的骑手，并按 `city`、`rank_in_city`、`courier_id` 升序排列。

> 本题口径：基准日取全局最新已送达日期；近 7 个自然日包含基准日，即 `datediff(基准日, delivered_ts 日期)` 取值为 0 到 6。

## 3. 标准答案

以下答案根据[公开题解](https://www.nowcoder.com/discuss/873357140038082560)的聚合与城市内排名思路改写为 Hive SQL，并按题目要求使用全局基准日及 7 个自然日窗口。

```sql
WITH data AS (
    SELECT stack(
        6,
        CAST(90001 AS BIGINT), 1, '2024-08-01 10:00:00', '2024-08-01 10:40:00', 45,
        CAST(90002 AS BIGINT), 1, '2024-08-03 09:00:00', '2024-08-03 10:10:00', 60,
        CAST(90003 AS BIGINT), 2, '2024-08-02 09:00:00', '2024-08-02 09:30:00', 40,
        CAST(90004 AS BIGINT), 2, '2024-08-05 08:30:00', '2024-08-05 09:15:00', 45,
        CAST(90005 AS BIGINT), 3, '2024-08-04 11:00:00', '2024-08-04 11:50:00', 60,
        CAST(80001 AS BIGINT), 1, '2024-07-20 09:00:00', '2024-07-20 10:00:00', 45
    ) AS (parcel_id, courier_id, shipped_ts, delivered_ts, promised_minutes)
),
courier AS (
    SELECT stack(
        3,
        1, 'Andy', 'Beijing', '2024-01-01',
        2, 'Ben', 'Beijing', '2024-02-01',
        3, 'Chen', 'Shanghai', '2024-03-01'
    ) AS (courier_id, courier_name, city, hired_at)
),
baseline AS (
    SELECT
        max(to_date(delivered_ts)) AS latest_delivered_date
    FROM data
),
deliveries_7d AS (
    SELECT
        p.parcel_id,
        p.courier_id,
        CAST(
            (
                unix_timestamp(p.delivered_ts, 'yyyy-MM-dd HH:mm:ss')
                - unix_timestamp(p.shipped_ts, 'yyyy-MM-dd HH:mm:ss')
            ) / 60 AS INT
        ) AS actual_minutes,
        p.promised_minutes
    FROM data p
    CROSS JOIN baseline b
    WHERE datediff(b.latest_delivered_date, to_date(p.delivered_ts)) BETWEEN 0 AND 6
),
courier_metrics AS (
    SELECT
        d.courier_id,
        c.courier_name,
        c.city,
        count(*) AS orders_7d,
        sum(CASE WHEN d.actual_minutes <= d.promised_minutes THEN 1 ELSE 0 END) AS on_time_7d,
        CAST(
            round(
                sum(CASE WHEN d.actual_minutes <= d.promised_minutes THEN 1 ELSE 0 END)
                * 1.0 / count(*),
                2
            ) AS DECIMAL(10, 2)
        ) AS on_time_rate,
        CAST(round(avg(d.actual_minutes), 2) AS DECIMAL(10, 2)) AS avg_minutes_7d
    FROM deliveries_7d d
    JOIN courier c
        ON d.courier_id = c.courier_id
    GROUP BY d.courier_id, c.courier_name, c.city
),
ranked_couriers AS (
    SELECT
        courier_id,
        courier_name,
        city,
        orders_7d,
        on_time_7d,
        on_time_rate,
        avg_minutes_7d,
        rank() OVER (
            PARTITION BY city
            ORDER BY on_time_rate DESC, avg_minutes_7d ASC
        ) AS rank_in_city
    FROM courier_metrics
)
SELECT
    courier_id,
    courier_name,
    city,
    orders_7d,
    on_time_7d,
    on_time_rate,
    avg_minutes_7d,
    rank_in_city
FROM ranked_couriers
WHERE orders_7d > 0
ORDER BY city, rank_in_city, courier_id;
```
