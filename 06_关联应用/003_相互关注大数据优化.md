# 相互关注大数据优化

## 1. 题目描述

给定表 `fans`，这是一张关注表，用来记录用户之间的关注动作，字段如下：

| 字段名 | 含义 |
|---|---|
| `from_user` | 关注用户 |
| `to_user` | 被关注用户 |

示例数据：

| from_user | to_user |
|---|---|
| A | B |
| A | C |
| A | D |
| B | A |
| B | E |
| C | A |

要求查询：当关注表数据量非常大时，不使用自连接，找出相互关注的用户对。

> 本题口径：把 `A -> B` 和 `B -> A` 归一成同一个无向用户对，只输出互关用户对，每对用户只保留一行。

## 2. 期望结果

| user1 | user2 |
|---|---|
| A | B |
| A | C |

## 3. 标准答案

```sql
with relation_data as (
    select 'A' as from_user, 'B' as to_user union all
    select 'A', 'C' union all
    select 'A', 'D' union all
    select 'B', 'A' union all
    select 'B', 'E' union all
    select 'C', 'A'
),
relation_distinct as (
    select distinct
        from_user,
        to_user
    from relation_data
),
pair_data as (
    select
        case
            when from_user < to_user then from_user
            else to_user
        end as user1,
        case
            when from_user < to_user then to_user
            else from_user
        end as user2
    from relation_distinct
)
select
    user1,
    user2
from pair_data
group by
    user1,
    user2
having count(*) = 2;
```

## 6. 我的复盘要点

- 【下次先想】数据量特别大时，互关题不一定要自连接，可以先把有方向关系转成无向用户对，再聚合计数。
- 【修正记法】`pair_data` 里把 `A -> B` 和 `B -> A` 都统一成同一个 key，例如都转成 `A, B`，这样 `group by user1, user2 having count(*) = 2` 就能找出互关。
- 【本题坑点】要先在 `relation_distinct` 对原始有向关注关系去重，再做无向归一；不能归一化后立刻去重，否则 `A -> B` 和 `B -> A` 会被折叠成一行，`count(*)` 到不了 2。
- 【本题坑点】这种优化版本适合输出“互关用户对”；如果题目要求保留每条原始关注记录并打 `is_friend` 标记，仍需要把互关结果再关联回原表。
