# 计算平均支付间隔

## 1. 题目描述

- 牛客题号：SQL62
- 难度：中等
- 题目来源：[支付间隔平均值](https://www.nowcoder.com/practice/847431ad931e45348eb1ab5657144c28)
- 题解依据：[题解｜支付间隔平均值](https://www.nowcoder.com/discuss/847891211250565120)

小明负责支付订单的日志分析工作，公司有 `order_log` 和 `select_log` 两张表。

`order_log` 表记录玩家创建订单的客户端日志，字段如下：

| 字段名 | 含义 |
|---|---|
| `order_id` | 订单号 |
| `uid` | 用户 id |
| `logtime` | 日志采集时间 |
| `time` | 客户端记录时间 |
| `product_id` | 商品 id |
| `pay_method` | 支付方式 |

示例数据：

| order_id | uid | logtime | time | product_id | pay_method |
|---|---|---|---|---|---|
| aaaa | user_0001 | 2021-01-01 10:00:00 | 2021-01-01 10:00:00 | p599 |  |
| bbbb | user_0006 | 2022-01-01 09:59:58 | 2021-01-01 09:59:58 | p599 |  |
| cccc | user_0006 | 2022-01-01 09:59:58 | 2021-01-01 09:59:58 | p599 |  |

`select_log` 表记录玩家选择支付方式的客户端日志，字段如下：

| 字段名 | 含义 |
|---|---|
| `order_id` | 订单号 |
| `uid` | 用户 id |
| `logtime` | 日志采集时间 |
| `time` | 客户端记录时间 |
| `product_id` | 商品 id |
| `pay_method` | 支付方式 |

示例数据：

| order_id | uid | logtime | time | product_id | pay_method |
|---|---|---|---|---|---|
| aaaa | user_0001 | 2021-01-01 10:01:00 | 2021-01-01 10:01:00 | p599 | vx |
| bbbb | user_0006 | 2022-01-01 10:00:58 | 2021-01-01 10:00:58 | p599 | alipay |
| cccc | user_0006 | 2022-01-01 10:00:58 | 2021-01-01 10:00:58 | p599 | meituan |

要求查询：按 `order_id` 关联两张表中对应的支付事件，计算两条日志的 `logtime` 相差秒数的绝对值，再求所有订单间隔的平均值，以整数形式返回，输出列名为 `gap`。

> 本题口径：`logtime` 可能局部乱序，因此先对时间差取绝对值；无需考虑字段为空的情况。

## 3. 标准答案

```sql
WITH data AS (
    SELECT stack(
        6,
        'order_log',  'aaaa', 'user_0001', '2021-01-01 10:00:00', '2021-01-01 10:00:00', 'p599', '',
        'order_log',  'bbbb', 'user_0006', '2022-01-01 09:59:58', '2021-01-01 09:59:58', 'p599', '',
        'order_log',  'cccc', 'user_0006', '2022-01-01 09:59:58', '2021-01-01 09:59:58', 'p599', '',
        'select_log', 'aaaa', 'user_0001', '2021-01-01 10:01:00', '2021-01-01 10:01:00', 'p599', 'vx',
        'select_log', 'bbbb', 'user_0006', '2022-01-01 10:00:58', '2021-01-01 10:00:58', 'p599', 'alipay',
        'select_log', 'cccc', 'user_0006', '2022-01-01 10:00:58', '2021-01-01 10:00:58', 'p599', 'meituan'
    ) AS (log_type, order_id, uid, logtime, client_time, product_id, pay_method)
),
order_log AS (
    SELECT
        order_id,
        uid,
        logtime,
        client_time,
        product_id,
        pay_method
    FROM data
    WHERE log_type = 'order_log'
),
select_log AS (
    SELECT
        order_id,
        uid,
        logtime,
        client_time,
        product_id,
        pay_method
    FROM data
    WHERE log_type = 'select_log'
),
matched_payment_events AS (
    SELECT
        o.order_id,
        ABS(
            UNIX_TIMESTAMP(s.logtime, 'yyyy-MM-dd HH:mm:ss')
            - UNIX_TIMESTAMP(o.logtime, 'yyyy-MM-dd HH:mm:ss')
        ) AS gap_seconds
    FROM order_log o
    JOIN select_log s
        ON o.order_id = s.order_id
)
SELECT
    CAST(AVG(gap_seconds) AS BIGINT) AS gap
FROM matched_payment_events;
```
