# 网易云音乐推荐

## 1. 题目描述

题目来源：牛客 SQL63（较难），[官方题目](https://www.nowcoder.com/practice/048ed413ac0e4cf4a774b906fc87e0e7)。公开解法依据：[牛客题解](https://www.nowcoder.com/discuss/859739221647454208)。
云音乐数据库包含 `follow`、`music_likes` 和 `music` 三张表。

`follow` 关注表：

| 字段名 | 类型 | 是否为空 | 含义 |
|---|---|---|---|
| `user_id` | `int(4)` | 否 | 关注人的用户 id |
| `follower_id` | `int(4)` | 否 | 被关注人的用户 id |

主键为 (`user_id`, `follower_id`)。

示例数据：

| user_id | follower_id |
|---:|---:|
| 1 | 2 |
| 1 | 4 |
| 2 | 3 |

`music_likes` 用户喜欢的音乐表：

| 字段名 | 类型 | 是否为空 | 含义 |
|---|---|---|---|
| `user_id` | `int(4)` | 否 | 用户 id |
| `music_id` | `int(4)` | 否 | 用户喜欢的音乐 id |

主键为 (`user_id`, `music_id`)。

示例数据：

| user_id | music_id |
|---:|---:|
| 1 | 17 |
| 2 | 18 |
| 2 | 19 |
| 3 | 20 |
| 4 | 17 |

`music` 音乐表：

| 字段名 | 类型 | 是否为空 | 含义 |
|---|---|---|---|
| `id` | `int(4)` | 否 | 音乐 id |
| `music_name` | `varchar(32)` | 否 | 音乐名称 |

主键为 `id`。

示例数据：

| id | music_name |
|---:|---|
| 17 | yueyawang |
| 18 | kong |
| 19 | MOM |
| 20 | Sold Out |

要求查询：向 `user_id = 1` 的用户推荐其关注的人喜欢的音乐。不要推荐该用户已经喜欢的音乐，返回结果不得包含重复项，并按音乐 id 升序排列。

## 3. 标准答案

```sql
WITH data AS (
    select stack(
        12,
        'follow',      1,  2, cast(null as string),
        'follow',      1,  4, cast(null as string),
        'follow',      2,  3, cast(null as string),
        'music_likes', 1, 17, cast(null as string),
        'music_likes', 2, 18, cast(null as string),
        'music_likes', 2, 19, cast(null as string),
        'music_likes', 3, 20, cast(null as string),
        'music_likes', 4, 17, cast(null as string),
        'music',      17, cast(null as int), 'yueyawang',
        'music',      18, cast(null as int), 'kong',
        'music',      19, cast(null as int), 'MOM',
        'music',      20, cast(null as int), 'Sold Out'
    ) as (source_table, first_id, second_id, music_name)
),
follow as (
    select
        first_id as user_id,
        second_id as follower_id
    from data
    where source_table = 'follow'
),
music_likes as (
    select
        first_id as user_id,
        second_id as music_id
    from data
    where source_table = 'music_likes'
),
music as (
    select
        first_id as id,
        music_name
    from data
    where source_table = 'music'
),
followee_music as (
    select distinct
        ml.music_id
    from follow f
    join music_likes ml
        on f.follower_id = ml.user_id
    where f.user_id = 1
),
user_music as (
    select distinct
        music_id
    from music_likes
    where user_id = 1
),
recommended_music as (
    select distinct
        fm.music_id
    from followee_music fm
    left join user_music um
        on fm.music_id = um.music_id
    where um.music_id is null
)
select
    m.music_name
from recommended_music rm
join music m
    on rm.music_id = m.id
order by m.id;
```
