# 相互关注共同好友

## 1. 题目描述

给定表 `fans`，这是一张关注表，用来记录用户之间的关注动作，字段如下：

| 字段名 | 含义 |
|---|---|
| `from_user` | 关注用户 |
| `to_user` | 被关注用户 |
| `timestamp` | 关注时间 |

示例数据：

| from_user | to_user | timestamp |
|---|---|---|
| A | B | 2022-11-28 12:12:12 |
| A | C | 2022-11-28 12:12:13 |
| A | D | 2022-11-28 12:12:14 |
| B | A | 2022-11-28 12:12:16 |
| B | E | 2022-11-28 12:12:16 |
| C | A | 2022-11-28 12:12:17 |

要求查询：基于每一条关注记录，判断是否存在反向关注。如果 `from_user` 关注了 `to_user`，同时 `to_user` 也关注了 `from_user`，则说明两者相互关注，输出 `is_friend = 1`，否则输出 `is_friend = 0`。

> 本题口径：保留 `fans` 表中的每一条原始关注记录，并对每条记录打是否互关的标记。

## 2. 期望结果

| from_user | to_user | is_friend |
|---|---|---:|
| A | B | 1 |
| A | C | 1 |
| A | D | 0 |
| B | A | 1 |
| B | E | 0 |
| C | A | 1 |

## 3. 标准答案

```sql
with data as (
    select stack(
        6,
        'A', 'B', '2022-11-28 12:12:12',
        'A', 'C', '2022-11-28 12:12:13',
        'A', 'D', '2022-11-28 12:12:14',
        'B', 'A', '2022-11-28 12:12:16',
        'B', 'E', '2022-11-28 12:12:16',
        'C', 'A', '2022-11-28 12:12:17'
    ) as (from_user, to_user, `timestamp`)
)
select
    t1.from_user,
    t1.to_user,
    if(t2.from_user is not null, 1, 0) as is_friend
from data t1
left join data t2
    on t1.from_user = t2.to_user
    and t1.to_user = t2.from_user
order by
    t1.from_user,
    t1.to_user;
```

## 4. 我的解答

```sql
with data as (
    select stack(
        6,
        'A', 'B', '2022-11-28 12:12:12',
        'A', 'C', '2022-11-28 12:12:13',
        'A', 'D', '2022-11-28 12:12:14',

        'C', 'A', '2022-11-28 12:12:17'
    ) as (from_user, to_user, `timestamp`)
)
select
    t1.from_user,
    t1.to_user,
    ifnull(t2.from_user,1,0) is_friend
from
    data t1
left join
    data t2
on t1.from_user = t2.to_user
and t1.to_user = t2.from_user
```

## 5. 我的解答正确版本

```sql
with data as (
    select stack(
        6,
        'A', 'B', '2022-11-28 12:12:12',
        'A', 'C', '2022-11-28 12:12:13',
        'A', 'D', '2022-11-28 12:12:14',
        'B', 'A', '2022-11-28 12:12:16',
        'B', 'E', '2022-11-28 12:12:16',
        'C', 'A', '2022-11-28 12:12:17'
    ) as (from_user, to_user, `timestamp`)
)
select
    t1.from_user,
    t1.to_user,
    if(t2.from_user is not null, 1, 0) as is_friend
from data t1
left join data t2
    on t1.from_user = t2.to_user
    and t1.to_user = t2.from_user
order by
    t1.from_user,
    t1.to_user;
```

## 6. 我的复盘要点

- 【下次先想】这题本质是同一张关注表做自连接，左表表示当前关注关系，右表用来找反向关注关系。
- 【修正记法】判断 `A -> B` 是否互关，要去表里找 `B -> A`，所以连接条件要交叉写成 `t1.from_user = t2.to_user and t1.to_user = t2.from_user`。
- 【错误原因】我原 SQL 的连接思路是对的，问题在 `ifnull(t2.from_user,1,0)`：`ifnull` 不是三参数条件判断，应该用 `if(t2.from_user is not null, 1, 0)`。
- 【本题坑点】题目要求保留每条原始关注记录并打标，所以要用 `left join`；如果用 `inner join`，`A -> D`、`B -> E` 这种单向关注会被过滤掉。
