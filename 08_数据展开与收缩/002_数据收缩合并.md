# 数据收缩合并

## 1. 题目描述

给定表 `a`，字段如下：

| 字段名 | 含义 |
|---|---|
| `id` | 原始行编号 |
| `name` | 待合并的名称 |

示例数据：

| id | name |
|---:|---|
| 1 | aa |
| 2 | aa |
| 3 | aa |
| 4 | d |
| 5 | c |
| 6 | aa |
| 7 | aa |
| 8 | e |
| 9 | f |
| 10 | g |

要求查询：

1. 从 A 表得到 B 表：把相同 `name` 的所有行全局合并，输出该组最大 `id`，并用 `|` 拼接该组的 `name`。
2. 从 A 表得到 C 表：只合并相邻连续且 `name` 相同的行，输出每个连续段的最大 `id`，并用 `|` 拼接该段的 `name`。

> 本题口径：B 表按 `name` 全局收缩；C 表按相邻连续段收缩，前后两段相同 `name` 中间被其他值打断时不能合并。

## 2. 期望结果

A 到 B：

| id | name |
|---:|---|
| 7 | aa\|aa\|aa\|aa\|aa |
| 4 | d |
| 5 | c |
| 8 | e |
| 9 | f |
| 10 | g |

A 到 C：

| id | name |
|---:|---|
| 3 | aa\|aa\|aa |
| 4 | d |
| 5 | c |
| 7 | aa\|aa |
| 8 | e |
| 9 | f |
| 10 | g |

## 3. 标准答案

子问题 1：A 到 B，相同 `name` 全局合并。

```sql
with a as (
    select stack(
        10,
        1, 'aa',
        2, 'aa',
        3, 'aa',
        4, 'd',
        5, 'c',
        6, 'aa',
        7, 'aa',
        8, 'e',
        9, 'f',
        10, 'g'
    ) as (id, name)
),
tagged as (
    select
        id,
        name,
        min(id) over (partition by name) as min_id,
        max(id) over (partition by name) as max_id
    from a
),
ordered_data as (
    select
        min_id,
        max_id,
        name
    from tagged
    order by min_id, id
)
select
    max_id as id,
    concat_ws('|', collect_list(name)) as name
from ordered_data
group by min_id, max_id
order by min_id;
```

子问题 2：A 到 C，只合并相邻连续的相同 `name`。

```sql
with a as (
    select stack(
        10,
        1, 'aa',
        2, 'aa',
        3, 'aa',
        4, 'd',
        5, 'c',
        6, 'aa',
        7, 'aa',
        8, 'e',
        9, 'f',
        10, 'g'
    ) as (id, name)
),
lag_data as (
    select
        id,
        name,
        lag(name, 1, name) over (order by id) as lag_name
    from a
),
group_data as (
    select
        id,
        name,
        sum(if(name != lag_name, 1, 0)) over (
            order by id
            rows between unbounded preceding and current row
        ) as grp
    from lag_data
),
tagged as (
    select
        id,
        name,
        grp,
        max(id) over (partition by grp) as max_id
    from group_data
),
ordered_data as (
    select
        max_id,
        grp,
        name
    from tagged
    order by grp, id
)
select
    max_id as id,
    concat_ws('|', collect_list(name)) as name
from ordered_data
group by max_id, grp
order by max_id;
```

## 4. 我的解答

子问题 1 的原始解答：

```sql
with a as (
    select stack(
        10,
        1, 'aa',
        2, 'aa',
        3, 'aa',
        4, 'd',
        5, 'c',
        6, 'aa',
        7, 'aa',
        8, 'e',
        9, 'f',
        10, 'g'
    ) as (id, name)
),
ordered_data as (
    select
        min(id) min_id,
        max(id) max_id,
        CONCAT_WS("|",COLLECT_LIST(name)) ws_name
    from a
    group by
        name
    order by
        min_id
)
select
    max_id,
    ws_name
from
    ordered_data;
```

子问题 2 的原始解答：

```sql
with a as (
    select stack(
        10,
        1, 'aa',
        2, 'aa',
        3, 'aa',
        4, 'd',
        5, 'c',
        6, 'aa',
        7, 'aa',
        8, 'e',
        9, 'f',
        10, 'g'
    ) as (id, name)
),
ordered_data as (
    select
        id,
        name,
        lag(name) over(order by id) order_name
    from
        a
),
sum_data as (
    select
        id,
        name,
        sum(
            case
                when order_name = name then 0
                when order_name is null then 0
                else 1
            end
        ) over(order by id) sum_data
    from
        ordered_data
)
select
    max(id) max_id,
    CONCAT_WS('|',COLLECT_LIST(name))
from
    sum_data
group by
    sum_data
order by
    max_id;
```

## 5. 我的解答正确版本

子问题 1 的修正版本：

```sql
with a as (
    select stack(
        10,
        1, 'aa',
        2, 'aa',
        3, 'aa',
        4, 'd',
        5, 'c',
        6, 'aa',
        7, 'aa',
        8, 'e',
        9, 'f',
        10, 'g'
    ) as (id, name)
),
ordered_data as (
    select
        min(id) as min_id,
        max(id) as max_id,
        concat_ws('|', collect_list(name)) as ws_name
    from a
    group by name
    order by min_id
)
select
    max_id as id,
    ws_name as name
from ordered_data
order by min_id;
```

子问题 2 的修正版本：

```sql
with a as (
    select stack(
        10,
        1, 'aa',
        2, 'aa',
        3, 'aa',
        4, 'd',
        5, 'c',
        6, 'aa',
        7, 'aa',
        8, 'e',
        9, 'f',
        10, 'g'
    ) as (id, name)
),
ordered_data as (
    select
        id,
        name,
        lag(name) over (order by id) as prev_name
    from a
),
group_data as (
    select
        id,
        name,
        sum(
            case
                when prev_name is null then 0
                when prev_name = name then 0
                else 1
            end
        ) over (
            order by id
            rows between unbounded preceding and current row
        ) as grp
    from ordered_data
),
ordered_group_data as (
    select
        id,
        name,
        grp
    from group_data
    order by id
)
select
    max(id) as id,
    concat_ws('|', collect_list(name)) as name
from ordered_group_data
group by grp
order by id;
```

## 6. 我的复盘要点

- 【我的卡点】A 到 B 是“按值全局收缩”，你的 `group by name` 思路是对的；`concat_ws('|', collect_list(name))` 负责把同一个 `name` 的多行收缩成一个拼接字符串。
- 【修正记法】A 到 B 输出的合并后 `id` 应该用该组的 `max(id)`，而 `min(id)` 主要用于恢复各组第一次出现的顺序，例如 `aa` 要排在 `d`、`c` 前面。
- 【我的卡点】A 到 C 不是按 `name` 全局分组，而是先用 `lag(name)` 判断当前行和上一行是否变化，再对变化标记累计求和生成连续段编号 `grp`；你的解法这个核心思路是对的。
- 【修正记法】最终收缩后的 `id` 应该取每个连续段内的 `max(id)`，因为结果表 C 的 `id` 表示这一段被合并后落在最后一条原始记录上的编号。
- 【本题坑点】`concat_ws('|', collect_list(name))` 解决的是“段内多行 name 拼成一个字符串”；真正决定哪些行属于同一段的是累计出来的 `grp`，不是 `name` 本身。
