# 日期连续区间合并

## 1. 题目描述

给定用户状态区间表 `data`，每一行表示某个用户在某段日期内处于某个状态或类别。

| 字段名 | 含义 |
|---|---|
| `type` | 状态或类别 |
| `name` | 用户名 |
| `start_date` | 区间开始日期 |
| `end_date` | 区间结束日期 |

示例数据：

| type | name | start_date | end_date |
|---|---|---|---|
| a | 小明 | 2023-01-01 | 2023-01-05 |
| a | 小明 | 2023-01-06 | 2023-01-07 |
| b | 小明 | 2023-01-10 | 2023-01-12 |
| b | 小明 | 2023-01-14 | 2023-01-16 |
| a | 小明 | 2023-01-18 | 2023-01-20 |

要求按时间顺序合并相邻且 `type + name` 相同的连续记录，并输出 `type`、`name`、合并后开始日 `start_day`、合并后结束日 `end_day`。

> 本题口径：这里的“连续”按相邻记录状态连续理解，不严格要求日期无断点。也就是说，只要同一个 `name` 下相邻记录的 `type` 相同，就可以合并；如果中间切换到其他 `type` 后又切回相同 `type`，不能跨段合并。

## 2. 期望结果

| type | name | start_day | end_day |
|---|---|---|---|
| a | 小明 | 01 | 07 |
| b | 小明 | 10 | 16 |
| a | 小明 | 18 | 20 |

## 3. 标准答案

```sql
with data as (
    select 'a' as `type`, '小明' as `name`, '2023-01-01' as start_date, '2023-01-05' as end_date union all
    select 'a' as `type`, '小明' as `name`, '2023-01-06' as start_date, '2023-01-07' as end_date union all
    select 'b' as `type`, '小明' as `name`, '2023-01-10' as start_date, '2023-01-12' as end_date union all
    select 'b' as `type`, '小明' as `name`, '2023-01-14' as start_date, '2023-01-16' as end_date union all
    select 'a' as `type`, '小明' as `name`, '2023-01-18' as start_date, '2023-01-20' as end_date
),
base as (
    select
        `type`,
        `name`,
        to_date(start_date) as start_date,
        to_date(end_date) as end_date
    from data
),
ordered_data as (
    select
        `type`,
        `name`,
        start_date,
        end_date,
        lag(`type`) over(partition by `name` order by start_date, end_date) as prev_type
    from base
),
marked_data as (
    select
        `type`,
        `name`,
        start_date,
        end_date,
        case
            when prev_type is null then 1
            when prev_type <> `type` then 1
            else 0
        end as is_new_group
    from ordered_data
),
grouped_data as (
    select
        `type`,
        `name`,
        start_date,
        end_date,
        sum(is_new_group) over(
            partition by `name`
            order by start_date, end_date
            rows between unbounded preceding and current row
        ) as group_id
    from marked_data
),
merged_data as (
    select
        `type`,
        `name`,
        min(start_date) as min_start_date,
        max(end_date) as max_end_date
    from grouped_data
    group by `name`, group_id, `type`
)
select
    `type`,
    `name`,
    date_format(min_start_date, 'dd') as start_day,
    date_format(max_end_date, 'dd') as end_day
from merged_data
order by `name`, start_day;
```

## 6. 我的复盘要点

- 【本题坑点】不能直接按 `type + name` 全局分组，否则前后两段相同状态会被错误合并，中间被其他状态打断后必须重新开段。
- 【修正记法】这题的连续段编号来自两步：先用 `lag(type)` 判断当前行是否开启新段，再对 `is_new_group` 做累计求和。
- 【我的卡点】`rows between unbounded preceding and current row` 表示从当前用户第一行累计到当前行，适合把“新段标记”累加成稳定的 `group_id`。
- 【本题坑点】当前口径是“相邻状态连续”而不是严格日期无断点；如果题目要求日期也连续，还要比较上一段 `end_date` 和当前 `start_date`。
- 【下次先想】字段名是 `type` 这类容易冲突的词时，先加反引号，避免 Hive 解析出错。
