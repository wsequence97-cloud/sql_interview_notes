# 日期连续区间合并

## 1. 题目描述

给定用户日期区间数据，每一行表示某个用户在某个类别下的一段日期区间。

| 字段名 | 含义 |
|---|---|
| `id` | 类别标识 |
| `name` | 用户名 |
| `start_date` | 区间开始日期 |
| `end_date` | 区间结束日期 |

示例数据：

| id | name | start_date | end_date |
|---|---|---|---|
| a | 小明 | 2023-01-01 | 2023-01-03 |
| a | 小明 | 2023-01-04 | 2023-01-07 |
| b | 小明 | 2023-01-08 | 2023-01-12 |
| b | 小明 | 2023-01-13 | 2023-01-16 |
| a | 小明 | 2023-01-17 | 2023-01-20 |

要求按照 `id + name` 分组，将日期首尾严格相接的区间合并，并输出类别、用户名、合并后的开始日和结束日。

> 本题口径：如果上一行的 `end_date + 1 天 = 当前行的 start_date`，则两个区间连续，应合并到同一组；否则说明日期有间断，当前行开启一个新组。

## 2. 期望结果

| id | name | start_day | end_day |
|---|---|---|---|
| a | 小明 | 01 | 07 |
| b | 小明 | 08 | 16 |
| a | 小明 | 17 | 20 |

## 3. 标准答案

```sql
with data as (
    select 'a' as id, '小明' as name, cast('2023-01-01' as date) as start_date, cast('2023-01-03' as date) as end_date union all
    select 'a' as id, '小明' as name, cast('2023-01-04' as date) as start_date, cast('2023-01-07' as date) as end_date union all
    select 'b' as id, '小明' as name, cast('2023-01-08' as date) as start_date, cast('2023-01-12' as date) as end_date union all
    select 'b' as id, '小明' as name, cast('2023-01-13' as date) as start_date, cast('2023-01-16' as date) as end_date union all
    select 'a' as id, '小明' as name, cast('2023-01-17' as date) as start_date, cast('2023-01-20' as date) as end_date
),
t1 as (
    select
        id,
        name,
        start_date,
        end_date,
        lag(end_date) over(
            partition by id, name
            order by start_date
        ) as lag_date,
        case
            when date_add(
                lag(end_date) over(
                    partition by id, name
                    order by start_date
                ),
                interval 1 day
            ) = start_date then 0
            else 1
        end as new_group_flag
    from data
),
t2 as (
    select
        id,
        name,
        start_date,
        end_date,
        sum(new_group_flag) over(
            partition by id, name
            order by start_date
            rows between unbounded preceding and current row
        ) as group_id
    from t1
)
select
    id,
    name,
    date_format(min(start_date), '%d') as start_day,
    date_format(max(end_date), '%d') as end_day
from t2
group by
    id,
    name,
    group_id
order by
    min(start_date),
    id,
    name;
```

## 6. 我的复盘要点

- 【CTE 建数】先用 `data` CTE 构造题目给出的五条测试数据，使示例和解答可以放在同一段 SQL 中阅读和运行。
- 【解题思路】按 `id + name` 分区并按 `start_date` 排序，再用 `lag(end_date)` 取得同一类别、同一用户的上一段结束日期。
- 【断点判断】如果 `上一段 end_date + 1 天 = 当前 start_date`，标记为 `0`，表示仍在原连续组；否则标记为 `1`，表示开启新组。每个分区的第一行没有上一行，也会标记为 `1`。
- 【分组方法】对 `new_group_flag` 做累计求和，累计值相同的记录属于同一个连续区间，累计值就是 `group_id`。
- 【区间合并】按 `id + name + group_id` 分组，使用 `min(start_date)` 和 `max(end_date)` 得到合并后的区间边界。
- 【本题坑点】不能只按 `id + name` 聚合，否则类别 `a` 的 `01~07` 和 `17~20` 会被错误合并；必须带上由日期断点生成的 `group_id`。
- 【与 007 的区别】003 使用 `lag(end_date)` 判断日期是否连续；007 使用 `lag(type)` 判断相邻状态是否相同，日期中间有断也可以合并。
