# 找出 UV 增长率连续超过 30% 的日期

## 1. 题目描述

给定网站每日指标表 `your_table`，每一行表示某一天的 PV 和 UV 数据。

| 字段名 | 含义 |
|---|---|
| `日期` | 统计日期 |
| `pv` | 当日 PV |
| `uv` | 当日 UV |

示例数据：

| 日期 | pv | uv |
|---|---:|---:|
| 2024-01-01 | 1 | 100 |
| 2024-01-02 | 1 | 130 |
| 2024-01-03 | 1 | 150 |
| 2024-01-04 | 1 | 200 |
| 2024-01-05 | 1 | 270 |
| 2024-01-06 | 1 | 310 |
| 2024-01-07 | 1 | 300 |
| 2024-01-08 | 1 | 400 |

要求找出 UV 增长率连续超过 30% 的日期，并输出日期、PV、UV、UV 增长率百分比。

> 本题口径：UV 增长率按 `(当天 uv - 前一天 uv) / 前一天 uv` 计算；“超过 30%”按 `> 0.3` 理解，不包含刚好等于 30%；“连续”指满足增长率超过 30% 的日期本身连续出现，连续段长度至少为 2 天。

## 2. 期望结果

| 日期 | pv | uv | uv增长率百分比 |
|---|---:|---:|---:|
| 2024-01-04 | 1 | 200 | 33.33 |
| 2024-01-05 | 1 | 270 | 35.00 |

## 3. 标准答案

```sql
with data as (
    select '2024-01-01' as `日期`, 1 as pv, 100 as uv union all
    select '2024-01-02' as `日期`, 1 as pv, 130 as uv union all
    select '2024-01-03' as `日期`, 1 as pv, 150 as uv union all
    select '2024-01-04' as `日期`, 1 as pv, 200 as uv union all
    select '2024-01-05' as `日期`, 1 as pv, 270 as uv union all
    select '2024-01-06' as `日期`, 1 as pv, 310 as uv union all
    select '2024-01-07' as `日期`, 1 as pv, 300 as uv union all
    select '2024-01-08' as `日期`, 1 as pv, 400 as uv
),
growth as (
    select
        to_date(`日期`) as dt,
        pv,
        uv,
        lag(uv) over(order by to_date(`日期`)) as prev_uv
    from data
),
hit as (
    select
        dt,
        pv,
        uv,
        (uv - prev_uv) / cast(prev_uv as double) as uv_growth_rate
    from growth
    where prev_uv is not null
      and (uv - prev_uv) / cast(prev_uv as double) > 0.3
),
grp as (
    select
        dt,
        pv,
        uv,
        uv_growth_rate,
        date_sub(dt, cast(row_number() over(order by dt) as int)) as grp_key
    from hit
)
select
    date_format(dt, 'yyyy-MM-dd') as `日期`,
    pv,
    uv,
    round(uv_growth_rate * 100, 2) as `uv增长率百分比`
from grp
where grp_key in (
    select
        grp_key
    from grp
    group by grp_key
    having count(*) >= 2
)
order by dt;
```

## 5. 我的解答正确版本

```sql
with data0 as (
    select '2024-01-01' as dt, 1 as pv, 100 as uv union all
    select '2024-01-02' as dt, 1 as pv, 130 as uv union all
    select '2024-01-03' as dt, 1 as pv, 150 as uv union all
    select '2024-01-04' as dt, 1 as pv, 200 as uv union all
    select '2024-01-05' as dt, 1 as pv, 270 as uv union all
    select '2024-01-06' as dt, 1 as pv, 310 as uv union all
    select '2024-01-07' as dt, 1 as pv, 300 as uv union all
    select '2024-01-08' as dt, 1 as pv, 400 as uv
),
base as (
    select 
        to_date(dt) as dt,
        pv,
        uv
    from data0
),
ordered_data as (
    select 
        dt,
        pv,
        uv,
        lag(uv) over(order by dt) as prev_uv
    from base
),
marked_data as (
    select 
        dt,
        pv,
        uv,
        prev_uv,
        (uv - prev_uv) / cast(prev_uv as double) as uv_growth_rate
    from ordered_data
    where prev_uv is not null
      and (uv - prev_uv) / cast(prev_uv as double) > 0.3
),
grp as (
    select
        dt,
        pv,
        uv,
        uv_growth_rate,
        date_sub(dt, cast(row_number() over(partition by pv order by dt) as int)) as sub_dt
    from marked_data
)
select 
    date_format(dt, 'yyyy-MM-dd') as dt,
    pv,
    uv,
    round(uv_growth_rate * 100, 2) as uv_growth_rate_percent
from grp
where sub_dt in (
    select 
        sub_dt
    from grp
    group by sub_dt
    having count(*) > 1
)
order by dt;
```

## 6. 我的复盘要点

- 【我的卡点】我已经先筛出了增长率超过 30% 的日期，这个方向是对的；但增长率计算要把分母转成 `double`，否则整数除法会丢小数。
- 【本题坑点】“超过 30%”是 `> 0.3`，不是 `>= 0.3`；2024-01-02 刚好 30%，不能算命中。
- 【修正记法】先用 `lag(uv)` 算前一天 UV，再筛命中日期，最后对命中日期用 `date_sub(dt, row_number)` 做连续分组。
- 【错误原因】最后不能写成 `(select count(*) from grp group by sub_dt) > 1`，因为子查询会返回多行；应该用 `sub_dt in (...) having count(*) > 1`。
- 【下次先想】连续命中类题目不是看所有日期连续，而是先筛出命中日期，再判断命中日期本身是否连续。
