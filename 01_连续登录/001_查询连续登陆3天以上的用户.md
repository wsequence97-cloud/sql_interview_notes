# 查询连续登录 3 天以上的用户

## 1. 题目描述

给定用户登录记录表 `data`，每一行表示某个用户的一次登录时间。

| 字段名 | 含义 |
|---|---|
| `id` | 用户 ID |
| `date` | 登录时间字符串 |

示例数据：

| id | date |
|---:|---|
| 1 | 2019-01-01 00:00:00 |
| 1 | 2019-01-02 00:00:00 |
| 1 | 2019-01-03 00:00:00 |
| 2 | 2019-02-01 00:00:00 |
| 2 | 2019-02-02 00:00:00 |
| 3 | 2019-03-04 00:00:00 |
| 3 | 2019-03-05 00:00:00 |
| 3 | 2019-03-06 00:00:00 |
| 3 | 2019-03-07 00:00:00 |

要求查询连续登录天数大于 3 天的用户，并输出用户 ID、连续登录分组标记日期 `date1`、连续登录天数 `day_cnt`。

> 本题口径：“3 天以上”按 `> 3` 理解，不包含刚好 3 天；如果题目要求“大于等于 3 天”，需要把 `having count(*) > 3` 改成 `having count(*) >= 3`。同一用户同一天可能多次登录，计算前需要按天去重。

## 2. 期望结果

| id | date1 | day_cnt |
|---:|---|---:|
| 3 | 2019-03-03 | 4 |

## 3. 标准答案

```sql
with data as (
    select 1 as id, '2019-01-01 00:00:00' as `date` union all
    select 1 as id, '2019-01-02 00:00:00' as `date` union all
    select 1 as id, '2019-01-03 00:00:00' as `date` union all
    select 2 as id, '2019-02-01 00:00:00' as `date` union all
    select 2 as id, '2019-02-02 00:00:00' as `date` union all
    select 3 as id, '2019-03-04 00:00:00' as `date` union all
    select 3 as id, '2019-03-05 00:00:00' as `date` union all
    select 3 as id, '2019-03-06 00:00:00' as `date` union all
    select 3 as id, '2019-03-07 00:00:00' as `date`
),
login_distinct as (
    select distinct
        id,
        to_date(`date`) as login_date
    from data
),
login_ranked as (
    select
        id,
        login_date,
        row_number() over(partition by id order by login_date) as rn
    from login_distinct
),
login_grouped as (
    select
        id,
        login_date,
        date_sub(login_date, cast(rn as int)) as date1
    from login_ranked
)
select
    id,
    date1,
    count(*) as day_cnt
from login_grouped
group by id, date1
having count(*) > 3
order by id, date1;
```

## 4. 我的解答

```sql
select
    id,
    count(date2)
from (
    select
        id,
        date_add(date, -row_number() over(partition by id order by date)) date2
    from data
) t1
group by id;
```

## 5. 我的解答正确版本

```sql
with login_distinct as (
    select distinct
        id,
        to_date(`date`) as login_date
    from data
),
login_grouped as (
    select
        id,
        date_sub(
            login_date,
            cast(row_number() over(partition by id order by login_date) as int)
        ) as date1
    from login_distinct
)
select
    id,
    date1,
    count(*) as day_cnt
from login_grouped
group by id, date1
having count(*) > 3
order by id, date1;
```

## 6. 我的复盘要点

- 【我的卡点】我一开始想到了“日期减行号”的连续登录套路，但最后只按 `id` 分组，会把同一个用户的多段连续登录混在一起。
- 【错误原因】连续段的分组键不是单独的 `id`，而是 `id + date_sub(login_date, rn)`；这个稳定日期才代表同一段连续登录。
- 【修正记法】连续登录题先按天去重，再 `row_number()`，再用 `date_sub(login_date, rn)` 生成连续段编号。
- 【本题坑点】“3 天以上”本题按 `> 3`，如果题目说“连续登录 3 天及以上”，才改成 `>= 3`。
