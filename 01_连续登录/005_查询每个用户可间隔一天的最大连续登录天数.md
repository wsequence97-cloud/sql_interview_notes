# 查询每个用户可间隔一天的最大连续登录天数

## 1. 题目描述

给定用户登录表 `test`，每一行表示某个用户在某一天登录过游戏。

| 字段名 | 含义 |
|---|---|
| `uid` | 用户 ID |
| `dt` | 登录日期 |

示例数据：

| uid | dt |
|---:|---|
| 1 | 2023-01-01 |
| 1 | 2023-01-03 |
| 1 | 2023-01-05 |
| 1 | 2023-01-06 |
| 1 | 2023-01-10 |
| 2 | 2023-02-01 |
| 2 | 2023-02-02 |
| 2 | 2023-02-04 |
| 2 | 2023-02-07 |
| 2 | 2023-02-08 |
| 3 | 2023-03-01 |
| 3 | 2023-03-04 |
| 3 | 2023-03-06 |
| 3 | 2023-03-08 |
| 4 | 2023-04-10 |

要求计算每个用户最大的连续登录天数，允许中间间隔一天。

> 本题口径：如果相邻两次登录日期相差不超过 2 天，则认为仍属于同一段连续登录。例如用户在 1、3、5、6 号登录，则视为从 1 号连续到 6 号，连续天数为 6 天。

## 2. 期望结果

| uid | day |
|---:|---:|
| 1 | 6 |
| 2 | 4 |
| 3 | 5 |
| 4 | 1 |

## 3. 标准答案

```sql
with data as (
    select 1 as uid, '2023-01-01' as dt union all
    select 1 as uid, '2023-01-03' as dt union all
    select 1 as uid, '2023-01-05' as dt union all
    select 1 as uid, '2023-01-06' as dt union all
    select 1 as uid, '2023-01-10' as dt union all
    select 2 as uid, '2023-02-01' as dt union all
    select 2 as uid, '2023-02-02' as dt union all
    select 2 as uid, '2023-02-04' as dt union all
    select 2 as uid, '2023-02-07' as dt union all
    select 2 as uid, '2023-02-08' as dt union all
    select 3 as uid, '2023-03-01' as dt union all
    select 3 as uid, '2023-03-04' as dt union all
    select 3 as uid, '2023-03-06' as dt union all
    select 3 as uid, '2023-03-08' as dt union all
    select 4 as uid, '2023-04-10' as dt
),
login_distinct as (
    select distinct
        uid,
        to_date(dt) as dt
    from data
),
ordered_data as (
    select
        uid,
        dt,
        lag(dt) over(partition by uid order by dt) as last_dt
    from login_distinct
),
marked_data as (
    select
        uid,
        dt,
        case
            when last_dt is null then 1
            when datediff(dt, last_dt) > 2 then 1
            else 0
        end as is_new_group
    from ordered_data
),
grouped_data as (
    select
        uid,
        dt,
        sum(is_new_group) over(
            partition by uid
            order by dt
            rows between unbounded preceding and current row
        ) as flag
    from marked_data
),
login_period as (
    select
        uid,
        flag,
        datediff(max(dt), min(dt)) + 1 as day_cnt
    from grouped_data
    group by uid, flag
)
select
    uid,
    max(day_cnt) as day
from login_period
group by uid
order by uid;
```

## 6. 我的复盘要点

- 【我的卡点】我一开始用 `count(*)` 算每段登录次数，但题目要的是连续日期跨度；比如 1、3、5、6 号登录，次数是 4，跨度天数是 6。
- 【修正记法】“允许间隔一天”的分段条件是 `datediff(dt, last_dt) <= 2` 仍算同一段，只有 `> 2` 才开启新段。
- 【错误原因】每段长度要用 `datediff(max(dt), min(dt)) + 1`，不能用该段实际登录记录数代替。
- 【本题坑点】累计出来的 `flag` 是连续段编号，计算每段跨度时必须按 `uid, flag` 分组，否则同一用户的多段会混在一起。
- 【下次先想】`datediff` 输出数字，`date_add/date_sub` 输出日期；做间隔判断用 `datediff`，做日期平移用 `date_add/date_sub`。
