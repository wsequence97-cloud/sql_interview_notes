# 日期有断区间合并

## 1. 题目描述

给定用户状态区间表 `data`，每一行表示某个用户在某段日期内处于某个状态或类别。当前场景中，部分相邻区间的日期中间有断开，但题目期望仍然按相邻状态段进行合并。

| 字段名 | 含义 |
|---|---|
| `type` | 状态或类别 |
| `name` | 用户名 |
| `start_date` | 区间开始日期 |
| `end_date` | 区间结束日期 |

示例数据：

| type | name | start_date | end_date |
|---|---|---|---|
| a | 小明 | 2023-01-01 | 2023-01-03 |
| a | 小明 | 2023-01-05 | 2023-01-07 |
| b | 小明 | 2023-01-10 | 2023-01-12 |
| b | 小明 | 2023-01-14 | 2023-01-16 |
| a | 小明 | 2023-01-18 | 2023-01-20 |

要求按时间顺序合并相邻且 `type + name` 相同的记录，并输出 `type`、`name`、合并后开始日 `start_day`、合并后结束日 `end_day`。

> 本题口径：日期中间是否断开不影响合并；只要相邻记录的 `type + name` 相同，就合并为同一段。

## 2. 期望结果

| type | name | start_day | end_day |
|---|---|---|---|
| a | 小明 | 01 | 07 |
| b | 小明 | 10 | 16 |
| a | 小明 | 18 | 20 |

## 3. 标准答案

```sql
with data as (
    select 'a' as `type`, '小明' as `name`, '2023-01-01' as start_date, '2023-01-03' as end_date union all
    select 'a' as `type`, '小明' as `name`, '2023-01-05' as start_date, '2023-01-07' as end_date union all
    select 'b' as `type`, '小明' as `name`, '2023-01-10' as start_date, '2023-01-12' as end_date union all
    select 'b' as `type`, '小明' as `name`, '2023-01-14' as start_date, '2023-01-16' as end_date union all
    select 'a' as `type`, '小明' as `name`, '2023-01-18' as start_date, '2023-01-20' as end_date
),
base as (
    select
        `type`,
        `name`,
        to_date(start_date) as start_date,
        to_date(end_date) as end_date
    from data
),
ordered_data as (
    select
        `type`,
        `name`,
        start_date,
        end_date,
        lag(`type`) over(
            partition by `name`
            order by start_date, end_date
        ) as prev_type
    from base
),
marked_data as (
    select
        `type`,
        `name`,
        start_date,
        end_date,
        case
            when prev_type is null then 1
            when prev_type <> `type` then 1
            else 0
        end as is_new_group
    from ordered_data
),
grouped_data as (
    select
        `type`,
        `name`,
        start_date,
        end_date,
        sum(is_new_group) over(
            partition by `name`
            order by start_date, end_date
            rows between unbounded preceding and current row
        ) as group_id
    from marked_data
),
merged_data as (
    select
        `type`,
        `name`,
        min(start_date) as min_start_date,
        max(end_date) as max_end_date
    from grouped_data
    group by `name`, group_id, `type`
)
select
    `type`,
    `name`,
    date_format(min_start_date, 'dd') as start_day,
    date_format(max_end_date, 'dd') as end_day
from merged_data
order by start_day, `type`, `name`;
```

## 6. 我的复盘要点

- 【我的卡点】这题容易被“日期中间有断”带偏，以为要判断 `end_date + 1 = start_date`；但图片给出的期望结果仍然把 `01-01~01-03` 和 `01-05~01-07` 合并了。
- 【错误原因】如果用 `date_add(lag(end_date), 1) = start_date` 做新段判断，第二种场景会输出 `01~03` 和 `05~07` 两段，不符合图片里的期望结果。
- 【修正记法】本题真正的打断条件是 `type` 变化，而不是日期断开；所以要用 `lag(type)` 判断相邻记录状态是否变化。
- 【本题坑点】窗口累计分组要按 `name` 分区，不能按 `type + name` 分区；否则后面再次出现的 `a` 可能和前面的 `a` 被放进同一条状态线里。
- 【下次先想】遇到区间合并题，先看期望结果到底按“日期连续”合并，还是按“相邻状态相同”合并，不能只看题目标题里的“日期连续”。
